# ADR 0001：使用编译期课程目录驱动渐进式学习

- 状态：已采纳
- 日期：2026-07-15

## 背景

项目原本以文章和题库浏览为中心，内容丰富但缺少明确次序、完成标准与复习闭环。目标用户已有基础编程能力，需要一条可以每天推进、又不阻止自由探索的路线。

## 决策

采用版本化的 `content/curriculum/catalog.json` 作为课程事实来源：

- 12 阶段、100 课、150 道精选练习、25 个动态图元数据；
- 目录由 `scripts/generate_curriculum.py` 根据文章 frontmatter 和本地题库确定性生成；
- Rust 启动时解析并校验唯一性、引用完整性和前置顺序；
- 课程前置关系是软约束，只向用户提示，不拒绝访问；
- 完成状态由服务端证据推导，不接受客户端直接把课程标为完成；
- 完成证据为小测最高分至少 80、核心题 Accepted，以及存在动态图时播放到最后一帧；
- 课程完成后创建 1/3/7/14/30 天复习计划。

文章正文继续作为教学内容，旧文章和完整题库保留在资料库，不复制一份课程专用正文。

## 理由

- 编译期目录让单二进制离线部署不依赖额外 CMS。
- 确定性生成避免手工维护大型 JSON，同时可以在测试中锁定数量和引用。
- 服务端推导完成状态可保持多个浏览器和导入数据的一致性。
- 软前置在提供学习建议的同时，允许有经验的用户跳读。

## 影响

优点：课程版本可审查、可测试、离线可用；学习状态规则集中；旧资料无需迁移或删除。

代价：更新课程需要重新生成目录并重新构建主站；文章质量仍需在源 Markdown 中维护；跨版本删除课程时必须提供进度迁移策略。

## 被否决方案

- 把课程完全存入 SQLite：部署可编辑性更高，但内容版本、引用审查和单二进制发布更复杂。
- 用前端 localStorage 推导完成状态：实现简单，但跨浏览器不一致，且容易产生无证据的“完成”。
- 硬锁前置课程：次序更强，但不适合已经掌握部分主题的用户。
