# 完全背包正序更新：动态规划训练题解

完全背包和 0/1 背包只差一句话：每种物品可以用无限次。这个差异直接决定容量遍历方向。

一句话记法：**完全背包正序容量，因为允许继续使用当前物品。**

## 适用场景

- 零钱兑换最少硬币数。
- 零钱兑换 II 方案数。
- 完全平方数。
- 元素可重复选择的组合问题。

如果每个元素只能用一次，就不是完全背包。

## Go 参考实现：零钱兑换

```go
func coinChange(coins []int, amount int) int {
	inf := amount + 1
	dp := make([]int, amount+1)
	for i := 1; i <= amount; i++ {
		dp[i] = inf
	}
	for _, c := range coins {
		for j := c; j <= amount; j++ {
			if dp[j-c]+1 < dp[j] {
				dp[j] = dp[j-c] + 1
			}
		}
	}
	if dp[amount] == inf {
		return -1
	}
	return dp[amount]
}
```

## 组合数和排列数

零钱兑换 II 求组合数，外层硬币、内层金额：

```go
dp[0] = 1
for _, c := range coins {
	for j := c; j <= amount; j++ {
		dp[j] += dp[j-c]
	}
}
```

组合总和 IV 求排列数，外层金额、内层数字：

```go
dp[0] = 1
for j := 1; j <= target; j++ {
	for _, x := range nums {
		if x <= j {
			dp[j] += dp[j-x]
		}
	}
}
```

## 为什么这样写

完全背包中，更新 `dp[j]` 时可以使用本轮刚更新过的 `dp[j-c]`，因为这代表当前硬币已经可以被选过一次，再选一次仍然合法。

循环顺序还会影响“方案是否区分顺序”。外层物品会固定物品使用顺序，所以算组合；外层容量会枚举最后一步选哪个物品，所以算排列。

## 复杂度

- 时间复杂度：$O(n \cdot amount)$。
- 空间复杂度：$O(amount)$。

## 易错点

- 完全背包容量倒序，导致每个硬币只能用一次。
- 最少硬币数中 `inf` 初始化太小。
- 方案数题把组合和排列循环顺序写反。
- 忘记 `dp[0] = 1` 或 `dp[0] = 0` 的语义区别。
