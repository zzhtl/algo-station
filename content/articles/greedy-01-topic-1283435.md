# 区间按结束时间排序：贪心训练题解

区间调度要选尽量多的不重叠区间。最可靠的贪心不是按开始时间，也不是按区间长度，而是按结束时间升序。

一句话记法：**越早结束，越不挡后面的区间。**

## 适用场景

- 选最多互不重叠区间。
- 删除最少区间让剩余不重叠。
- 区间链最长。
- 每次选择只影响“后面还能接什么”。

## Go 参考实现：无重叠区间

```go
func eraseOverlapIntervals(intervals [][]int) int {
	sort.Slice(intervals, func(i, j int) bool {
		return intervals[i][1] < intervals[j][1]
	})
	keep := 0
	end := math.MinInt
	for _, in := range intervals {
		if in[0] >= end {
			keep++
			end = in[1]
		}
	}
	return len(intervals) - keep
}
```

## 为什么这样写

假设最优解第一个选的是区间 `X`，贪心选的是结束时间最早的 `Y`。因为 `Y.end <= X.end`，用 `Y` 替换 `X` 不会和后续区间产生更多冲突，选出的区间数量不变。因此存在一个以贪心选择开头的最优解。

重复这个交换论证，按右端点排序的贪心成立。

## 复杂度

- 时间复杂度：$O(n \log n)$。
- 空间复杂度：取决于排序实现。

## 易错点

- 按左端点排序，局部很早开始的长区间可能挡住很多短区间。
- 重叠边界没按题意处理，`[1,2]` 和 `[2,3]` 通常不重叠。
- 题目要求删除数，代码算的是保留数，最后忘记转换。
- Go 需要引入 `sort` 和 `math`。
