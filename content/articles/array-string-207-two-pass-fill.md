# 两次扫描填充左右约束：数组与字符串精讲

## 一句话总结

一个方向的扫描只能看见单侧信息，两次扫描后再合并两侧约束。

## 识别信号

分糖果、除自身乘积、接雨水、字符最短距离、移动球。

对「两次扫描填充左右约束」来说，识别信号不是关键词本身，而是它们都在要求你在一段连续结构里保留足够信息，让窗口、指针或扫描线每推进一步都能解释答案为什么没有被漏掉。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 指针与区间锚点

- 本篇围绕「两次扫描填充左右约束」训练：一个方向的扫描只能看见单侧信息，两次扫描后再合并两侧约束。
- 数组题的核心不是多写循环，而是让每个下标只承担一种角色：左边界、右边界、候选值或答案落点。
- 识别信号里的“分糖果、除自身乘积、接雨水、字符最短距离、移动球”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #135、#238、#42、#821、#1769 时，复盘「边界移动、候选淘汰、答案写入」是否还能用同一套解释串起来。

## 推导步骤

1. 左到右记录左侧贡献。
2. 右到左记录右侧贡献。
3. 按题意取 max、乘积或求和。
4. 检查边界默认值。

## 动手前的边界清单

- 空数组、单元素、全相等、严格递增或递减都要先手算一遍。
- 对照「两次扫描填充左右约束」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #135、#238、#42、#821、#1769 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 合并方式用错。
- 第二遍覆盖第一遍信息。
- 边界位置没有合适的初始贡献。
