# 递归返回值契约：二叉树精讲

## 一句话总结

每个节点只向父节点返回可继续向上使用的信息，答案可在过程中全局更新。

## 识别信号

高度、平衡树、直径、最长同值路径。

对「递归返回值契约」来说，识别信号不是关键词本身，而是它们都在要求你把整棵树的问题拆成当前节点能向父节点交付什么，以及当前节点如何合并左右子树的信息。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 递归返回值锚点

- 本篇围绕「递归返回值契约」训练：每个节点只向父节点返回可继续向上使用的信息，答案可在过程中全局更新。
- 递归函数的返回值必须只表达对子树已经完成的承诺，不能夹带依赖父节点才知道的答案。
- 识别信号里的“高度、平衡树、直径、最长同值路径”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #104、#110、#543、#687 时，复盘「返回值含义、全局答案更新点、空节点处理」是否还能用同一套解释串起来。

## 推导步骤

1. 定义返回值。
2. 递归左右子树。
3. 用左右返回值更新当前答案。
4. 返回父节点需要的信息。

## 写递归前的边界清单

- 空树、单节点、只有左子树、只有右子树、重复值都要单独过。
- 对照「递归返回值契约」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #104、#110、#543、#687 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 把全局答案当返回值。
- 直径边数节点数混淆。
- 空节点返回值不统一。
