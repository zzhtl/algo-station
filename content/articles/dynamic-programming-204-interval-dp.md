# 区间 DP 枚举长度：动态规划精讲

## 一句话总结

大区间依赖更小区间，先枚举长度能保证转移来源已计算。

## 识别信号

戳气球、多边形三角剖分、切棍子、猜数字。

对「区间 DP 枚举长度」来说，识别信号不是关键词本身，而是它们都在要求你把重复子问题压成状态，并保证每一次转移只从已经定义清楚的旧状态来。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 状态转移锚点

- 本篇围绕「区间 DP 枚举长度」训练：大区间依赖更小区间，先枚举长度能保证转移来源已计算。
- 状态含义要先于代码确定；如果一句话说不清 dp[i] 表示什么，转移大概率会写散。
- 识别信号里的“戳气球、多边形三角剖分、切棍子、猜数字”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #312、#1039、#1547、#375 时，复盘「状态定义、转移来源、遍历顺序」是否还能用同一套解释串起来。

## 推导步骤

1. 定义区间状态。
2. 枚举区间长度。
3. 枚举中间断点 k。
4. 合并左右区间和当前代价。

## 写转移前的边界清单

- 初始状态、不可能状态、容量或下标为 0、答案取 max 还是求和要分清。
- 对照「区间 DP 枚举长度」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #312、#1039、#1547、#375 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 开闭区间不统一。
- 断点范围漏端点。
- 没有添加虚拟边界。
