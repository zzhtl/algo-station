# 补数计数统计配对：哈希与前缀精讲

## 一句话总结

当前元素能和之前多少元素成对，取决于补数已经出现多少次。

## 识别信号

两数之和、K 和数对、好餐计数、歌曲时长整除。

对「补数计数统计配对」来说，识别信号不是关键词本身，而是它们都在要求你把重复查询改成一次记录，让“以前见过什么”和“当前还缺什么”可以 O(1) 对上。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 计数与映射锚点

- 本篇围绕「补数计数统计配对」训练：当前元素能和之前多少元素成对，取决于补数已经出现多少次。
- 哈希表里保存的必须是可复用的证据，而不是碰巧能过样例的中间值。
- 识别信号里的“两数之和、K 和数对、好餐计数、歌曲时长整除”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #1、#1679、#1711、#1010 时，复盘「键的定义、值的含义、首次或最新出现策略」是否还能用同一套解释串起来。

## 推导步骤

1. 遍历元素。
2. 计算补数。
3. 先用补数频次累加答案。
4. 再加入当前元素。

## 建表前的边界清单

- 重复值、负数、前缀为 0、同一个键多次出现要单独验。
- 对照「补数计数统计配对」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #1、#1679、#1711、#1010 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 先加入当前导致自己配自己。
- 取模补数写成 `k-x%k` 没再取模。
- 重复值组合数漏算。
