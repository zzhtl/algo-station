# 线性 DP 状态机：动态规划精讲

## 一句话总结

每个状态表示处理到当前位置后的合法处境，转移表示今天采取的动作。

## 识别信号

股票买卖、冷冻期、手续费、打家劫舍变体。

对「线性 DP 状态机」来说，识别信号不是关键词本身，而是它们都在要求你把重复子问题压成状态，并保证每一次转移只从已经定义清楚的旧状态来。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 状态转移锚点

- 本篇围绕「线性 DP 状态机」训练：每个状态表示处理到当前位置后的合法处境，转移表示今天采取的动作。
- 状态含义要先于代码确定；如果一句话说不清 dp[i] 表示什么，转移大概率会写散。
- 识别信号里的“股票买卖、冷冻期、手续费、打家劫舍变体”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #121、#122、#309、#714 时，复盘「状态定义、转移来源、遍历顺序」是否还能用同一套解释串起来。

## 推导步骤

1. 列出状态。
2. 枚举动作。
3. 从昨天状态转到今天状态。
4. 初始化不可能状态为负无穷。

## 写转移前的边界清单

- 初始状态、不可能状态、容量或下标为 0、答案取 max 还是求和要分清。
- 对照「线性 DP 状态机」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #121、#122、#309、#714 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 状态含义随循环变化。
- 同一天买入卖出不符合约束。
- 初值把不可能状态设成 0。
