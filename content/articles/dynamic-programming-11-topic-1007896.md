# 打家劫舍选与不选：动态规划训练题解

打家劫舍是线性 DP 的经典题。每个位置只有两种决策：偷当前房子，或者不偷当前房子。

一句话记法：**偷当前，看前两个；不偷当前，看前一个。**

## 状态定义

```text
dp[i] = 处理到第 i 间房时能偷到的最大金额
```

转移：

```text
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

## Go 参考实现

```go
func rob(nums []int) int {
	prev2, prev1 := 0, 0
	for _, x := range nums {
		cur := prev1
		if prev2+x > cur {
			cur = prev2 + x
		}
		prev2, prev1 = prev1, cur
	}
	return prev1
}
```

## 环形房子怎么处理

#213 首尾相邻，不能同时偷。拆成两个线性问题：

- 偷的范围是 `[0, n-2]`。
- 偷的范围是 `[1, n-1]`。

答案取两者最大值。

## 树形房子怎么处理

#337 不是线性相邻，而是父子相邻不能同时偷。每个节点返回两个状态：

```text
robThis = node.Val + leftSkip + rightSkip
skipThis = max(leftRob,leftSkip) + max(rightRob,rightSkip)
```

## 为什么这样写

当前房子如果偷，上一间不能偷，所以只能接 `dp[i-2]`。如果不偷当前，最优值就是处理到上一间的答案 `dp[i-1]`。

状态只依赖前两个，所以可以滚动到两个变量。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(1)$。

## 易错点

- 环形版本直接套线性版本，可能同时选首尾。
- 初始化写复杂导致空数组、单元素出错。
- 树形版本只返回一个值，父节点不知道子节点是否被偷。
