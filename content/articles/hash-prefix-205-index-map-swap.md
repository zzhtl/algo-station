# 下标映射支持 O(1) 删除：哈希与前缀精讲

## 一句话总结

删除中间元素贵，删除末尾便宜；把待删元素和末尾交换即可把成本降到 O(1)。

## 识别信号

随机集合、黑名单映射、频繁插入删除取随机。

对「下标映射支持 O(1) 删除」来说，识别信号不是关键词本身，而是它们都在要求你把重复查询改成一次记录，让“以前见过什么”和“当前还缺什么”可以 O(1) 对上。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 计数与映射锚点

- 本篇围绕「下标映射支持 O(1) 删除」训练：删除中间元素贵，删除末尾便宜；把待删元素和末尾交换即可把成本降到 O(1)。
- 哈希表里保存的必须是可复用的证据，而不是碰巧能过样例的中间值。
- 识别信号里的“随机集合、黑名单映射、频繁插入删除取随机”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #380、#381、#710 时，复盘「键的定义、值的含义、首次或最新出现策略」是否还能用同一套解释串起来。

## 推导步骤

1. 哈希表记录值到下标。
2. 删除时取末尾元素覆盖待删位置。
3. 更新末尾元素下标。
4. 弹出末尾。

## 建表前的边界清单

- 重复值、负数、前缀为 0、同一个键多次出现要单独验。
- 对照「下标映射支持 O(1) 删除」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #380、#381、#710 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 重复元素集合没有保存下标集合。
- 删除最后一个元素时重复更新。
- 随机范围映射边界错。
