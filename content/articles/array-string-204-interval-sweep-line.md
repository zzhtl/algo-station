# 差分扫描线处理区间事件：数组与字符串精讲

## 一句话总结

区间影响可以延迟到扫描时统一生效，事件点排序后覆盖数只在事件点变化。

## 识别信号

航班预订、拼车、会议室、日程重叠、区间覆盖判断。

对「差分扫描线处理区间事件」来说，识别信号不是关键词本身，而是它们都在要求你在一段连续结构里保留足够信息，让窗口、指针或扫描线每推进一步都能解释答案为什么没有被漏掉。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 指针与区间锚点

- 本篇围绕「差分扫描线处理区间事件」训练：区间影响可以延迟到扫描时统一生效，事件点排序后覆盖数只在事件点变化。
- 数组题的核心不是多写循环，而是让每个下标只承担一种角色：左边界、右边界、候选值或答案落点。
- 识别信号里的“航班预订、拼车、会议室、日程重叠、区间覆盖判断”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #1094、#1109、#1893、#253、#732 时，复盘「边界移动、候选淘汰、答案写入」是否还能用同一套解释串起来。

## 推导步骤

1. 把每个区间拆成进入和离开事件。
2. 统一开闭区间。
3. 按坐标排序累计。
4. 在关键点检查容量或最大值。

## 动手前的边界清单

- 空数组、单元素、全相等、严格递增或递减都要先手算一遍。
- 对照「差分扫描线处理区间事件」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #1094、#1109、#1893、#253、#732 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 闭区间和半开区间混用。
- 离开事件顺序错导致端点相接被算重叠。
- 坐标大时不用事件表而开巨大数组。
