# 坐标离散化建索引：哈希与前缀训练题解

值域很大时不能直接开数组。离散化把实际出现过的值映射到 `0..m-1` 的连续下标。

## 手写步骤

1. 收集所有会查询或更新的值。
2. 排序并去重。
3. 用二分或哈希表把原值映射到压缩下标。
4. 在压缩下标上使用树状数组、线段树或计数数组。

## Go 参考片段

```go
func compress(vals []int) ([]int, map[int]int) {
	sort.Ints(vals)
	uniq := []int{}
	for _, x := range vals {
		if len(uniq) == 0 || uniq[len(uniq)-1] != x {
			uniq = append(uniq, x)
		}
	}
	id := map[int]int{}
	for i, x := range uniq {
		id[x] = i
	}
	return uniq, id
}
```

## 为什么这样写

很多题只关心大小顺序或相等关系，不关心值本身的绝对大小。比如值可能到 `1e9`，但数组只有 `1e5` 个数，离散化后就能用 `1e5` 大小的数据结构。

如果需要查询 `<= x` 的数量，通常用排序数组二分得到位置，而不是只靠 map 精确查值。

## 易错点

- 只收集原值，漏掉查询边界值。
- 去重前没有排序。
- 需要保持大小顺序，却用普通 map 随机编号。
- 下标从 0 还是 1 没和树状数组约定一致。
