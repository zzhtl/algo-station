# 边界出发反向标记：搜索与图论训练题解

网格题如果从每个内部格子出发判断能否到边界，往往重复搜索。反过来，从边界出发标记哪些格子能连到边界，思路更清楚。

一句话记法：**正向难判断，就从终点集合反向扩散。**

## 适用场景

- #130：从边界 `O` 出发标记不会被翻转的区域。
- 太平洋大西洋水流：从海岸反向走到更高或等高格子。
- 多个目标点反推可达区域。

## Go 参考思路：边界连通区域

```go
func solve(board [][]byte) {
	m, n := len(board), len(board[0])
	var dfs func(int, int)
	dfs = func(r, c int) {
		if r < 0 || r >= m || c < 0 || c >= n || board[r][c] != 'O' {
			return
		}
		board[r][c] = '#'
		dfs(r+1, c)
		dfs(r-1, c)
		dfs(r, c+1)
		dfs(r, c-1)
	}
	for r := 0; r < m; r++ {
		dfs(r, 0)
		dfs(r, n-1)
	}
	for c := 0; c < n; c++ {
		dfs(0, c)
		dfs(m-1, c)
	}
	for r := 0; r < m; r++ {
		for c := 0; c < n; c++ {
			if board[r][c] == 'O' {
				board[r][c] = 'X'
			} else if board[r][c] == '#' {
				board[r][c] = 'O'
			}
		}
	}
}
```

## 为什么这样写

#130 中，和边界连通的 `O` 不会被翻转。与其判断每个 `O` 是否能走到边界，不如直接从边界 `O` 出发，把所有安全区域标出来。

太平洋大西洋水流也是同理：水从高到低流向海洋，反向搜索就从海洋边界出发，只能走向更高或等高的格子。

## 复杂度

- 时间复杂度：$O(mn)$。
- 空间复杂度：DFS 栈或 BFS 队列 $O(mn)$。

## 易错点

- 从内部每个点出发重复搜索。
- 反向搜索的高度条件写反。
- 边界行列重复访问没有问题，但要靠标记避免无限递归。
- 临时标记最后忘记恢复。
