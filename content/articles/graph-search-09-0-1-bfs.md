# 0-1 BFS 用双端队列：搜索与图论训练题解

0-1 BFS 是 Dijkstra 的特化版。边权只有 0 和 1 时，用双端队列就能维护距离从小到大的处理顺序。

一句话记法：**0 代价走队头，1 代价走队尾。**

## 适用场景

- 边权只有 0/1。
- 求单源最短路。
- 网格中改变方向代价为 1，不改变为 0。
- 移除障碍代价为 1，空地代价为 0。

边权超过 1 时，回到 Dijkstra。

## 核心逻辑

```text
如果 nd < dist[v]:
    dist[v] = nd
    w == 0 -> push_front(v)
    w == 1 -> push_back(v)
```

双端队列中的状态大致按距离非递减排列。

## 为什么这样写

Dijkstra 用堆保证每次取最小距离。0-1 BFS 中，新距离只可能是当前距离 `d` 或 `d+1`。距离为 `d` 的新状态应该优先处理，所以放队头；距离为 `d+1` 的状态放队尾。

这让复杂度从堆的 `log` 降到线性。

## 复杂度

- 时间复杂度：$O(V+E)$。
- 空间复杂度：$O(V+E)$。

## 易错点

- 看到网格最短路就用普通 BFS，忽略 0/1 权重。
- 权重 0 和 1 的入队方向写反。
- 不维护 `dist`，只用 visited，导致更短路径无法更新。
- 边权不是 0/1 仍使用双端队列。
