# 一维滚动数组压空间：动态规划训练题解

DP 表不一定都要完整保存。如果当前行只依赖上一行和当前行左侧，就可以用一维数组滚动压空间。

一句话记法：**能不能压缩，看依赖；怎么遍历，看会不会覆盖还要用的旧值。**

## 适用场景

- `f[i]` 只依赖前几个状态，比如爬楼梯、打家劫舍。
- `f[i][j]` 只依赖上一行同列、上一行左侧或本行左侧。
- 只需要最终答案，不需要还原完整路径。

如果后续要输出具体方案，完整表或额外决策记录可能更合适。

## 图解思路

```mermaid
flowchart LR
  A["二维 f[i][j]"] --> B{"只依赖上一行/左侧?"}
  B -->|是| C[压成一维 f[j]]
  B -->|否| D[保留必要维度]
  C --> E[决定 j 正序还是倒序]
```

状态压缩后的难点通常不是公式，而是遍历方向。

## 不变量

- `f[j]` 在更新前表示上一行的值。
- 更新后表示当前行的值。
- 如果转移需要上一行的 `f[j-w]`，容量必须倒序。
- 如果转移允许使用当前行的 `f[j-w]`，容量可以正序。

## Go 参考实现：打家劫舍

```go
func rob(nums []int) int {
	prev2, prev1 := 0, 0
	for _, x := range nums {
		cur := prev1
		if prev2+x > cur {
			cur = prev2 + x
		}
		prev2, prev1 = prev1, cur
	}
	return prev1
}
```

## Go 参考实现：不同路径

```go
func uniquePaths(m int, n int) int {
	f := make([]int, n)
	for j := 0; j < n; j++ {
		f[j] = 1
	}
	for i := 1; i < m; i++ {
		for j := 1; j < n; j++ {
			f[j] += f[j-1]
		}
	}
	return f[n-1]
}
```

## 为什么这样写

不同路径中，二维转移是：

```text
dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

压缩后，更新前的 `f[j]` 是上一行同列，`f[j-1]` 是当前行左侧，所以从左到右更新正好成立。

打家劫舍只依赖前两个状态，所以连数组都不需要，两个变量就够。

## 复杂度

- 时间复杂度通常不变。
- 空间复杂度从 $O(mn)$ 降到 $O(n)$，或从 $O(n)$ 降到 $O(1)$。

## 易错点

- 还没分析依赖就压缩，导致读到被覆盖的新值。
- 0/1 背包容量正序更新，把同一物品用了多次。
- 压缩后边界初始化忘改。
- 需要还原路径时把完整信息丢掉。
