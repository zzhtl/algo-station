# 逆波兰表达式即时求值：栈与队列训练题解

逆波兰表达式把运算符放在操作数后面，所以不需要括号和优先级解析。栈顶永远保存最近的可用操作数。

一句话记法：**数字入栈，符号弹两个，算完再入栈。**

## 适用场景

- 后缀表达式求值。
- 运算符都是二元运算。
- 表达式本身已经按优先级排好。

## 不变量

- 栈中保存尚未参与最终合并的中间结果。
- 遇到数字，直接压栈。
- 遇到运算符，弹出的第二个数是右操作数，第一个数是左操作数。
- 扫描结束栈中唯一元素就是答案。

## Go 参考实现

```go
func evalRPN(tokens []string) int {
	st := []int{}
	for _, t := range tokens {
		switch t {
		case "+", "-", "*", "/":
			b := st[len(st)-1]
			a := st[len(st)-2]
			st = st[:len(st)-2]
			switch t {
			case "+":
				st = append(st, a+b)
			case "-":
				st = append(st, a-b)
			case "*":
				st = append(st, a*b)
			case "/":
				st = append(st, a/b)
			}
		default:
			x, _ := strconv.Atoi(t)
			st = append(st, x)
		}
	}
	return st[0]
}
```

## 为什么这样写

表达式 `["2","1","+","3","*"]` 表示 `(2+1)*3`。扫描到 `+` 时，最近两个操作数 `2` 和 `1` 已经在栈顶，可以立即合并为 `3`。后面的 `*` 再合并 `3` 和 `3`。

减法和除法有顺序，必须先弹右操作数 `b`，再取左操作数 `a`，计算 `a-b`、`a/b`。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(n)$。

## 易错点

- 减法/除法操作数顺序反了。
- 负数字符串 `"-11"` 被误判成运算符。
- Go 代码需要引入 `strconv`。
- 除法按题目要求向零截断，Go/Rust 整数除法正好满足。
