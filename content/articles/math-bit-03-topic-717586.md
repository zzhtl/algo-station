# 摩尔投票找多数：数学与位运算训练题解

多数元素指出现次数超过 `n/2` 的元素。摩尔投票的核心是“不同元素两两抵消”：每删掉一个多数元素和一个非多数元素，多数元素仍然还是剩余数组里的多数。

## 状态定义

维护：

- `candidate`：当前幸存候选。
- `count`：候选相对其他元素的净票数。

遍历元素 `x`：

- `count == 0`：当前没有候选，把 `x` 设为候选。
- `x == candidate`：候选多一票。
- `x != candidate`：用一个不同元素抵消候选一票。

## Go 参考实现

```go
package main

func majorityElement(nums []int) int {
	candidate := 0
	count := 0

	for _, x := range nums {
		if count == 0 {
			candidate = x
		}
		if x == candidate {
			count++
		} else {
			count--
		}
	}

	return candidate
}
```

## 为什么最后剩下的一定是多数

因为多数元素数量超过一半，非多数元素全部拿来和它抵消，也抵不完。摩尔投票只是在线模拟了这个抵消过程，不需要真的删除元素。

如果题目不保证多数元素存在，就必须第二遍验证候选出现次数是否超过阈值。

## 易错点

- `count == 0` 时先换候选，再处理当前元素这一票。
- 这题不是找出现次数最多的元素，而是找超过一半的元素；没有这个保证时不能直接返回。
- 多数阈值如果变成 `n/3`，需要维护两个候选，不是简单改条件。

## 复杂度

时间 `O(n)`，空间 `O(1)`。
