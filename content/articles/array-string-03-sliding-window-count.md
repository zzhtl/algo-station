# 滑动窗口维护计数：数组与字符串训练题解

滑动窗口解决的是连续区间问题。它的核心不是“右指针一直走，左指针偶尔走”，而是维护一个窗口不变量：**`[left, right]` 里的统计信息能立刻回答当前窗口是否合法、是否能更新答案**。

一旦你需要反复重新统计窗口内容，滑动窗口的优势就没了。所以这类题通常要配一个 `count`、`sum`、`distinct` 或 `valid` 变量。

## 适用场景

题目要求的是连续子数组、连续子串，并且窗口扩大或缩小时，状态能用 $O(1)$ 或接近 $O(1)$ 的代价更新。

- 求最长：窗口非法时收缩，合法时更新最大长度。
- 求最短：窗口满足条件后尽量收缩，每次收缩前更新最短长度。
- 判断排列或异位词：窗口长度通常固定，计数表匹配时记录答案。
- 替换、覆盖、包含类问题：除了计数表，还要维护当前窗口中最关键的计数，如最大频次或满足条件的字符种类数。

如果区间不是连续的，或者删除中间元素会影响最优结构，就不要硬套滑动窗口。

## 图解思路

```mermaid
flowchart LR
  A[right 右移纳入新元素] --> B[更新 sum/count/valid]
  B --> C{窗口是否需要收缩?}
  C -->|需要| D[移出 nums[left]]
  D --> E[left += 1]
  E --> C
  C -->|不需要| F[按题意更新答案]
  F --> A
```

滑动窗口最重要的是先决定“收缩条件”：

- 最长无重复子串：当某个字符出现次数大于 1 时收缩。
- 长度最小的子数组：当窗口和已经大于等于目标时，先更新答案再收缩。
- 找所有异位词：窗口长度超过模式串长度时收缩，长度相等时检查计数是否匹配。

同样是滑动窗口，答案更新时机并不一样。先写清楚“合法窗口是什么”，再决定是在 `while` 前、`while` 中，还是 `while` 后更新答案。

## 手写步骤

1. 初始化 `left = 0` 和答案。
2. 用 `right` 扫描，每次把新元素加入窗口统计。
3. 根据题目定义判断窗口是否需要收缩。
4. 每移出 `nums[left]`，同步撤销它对统计变量的影响。
5. 在正确时机更新答案：最长题通常在收缩后，最短题通常在满足条件时边收缩边更新。

## Go 参考骨架

```go
func lengthOfLongestSubstring(s string) int {
	cnt := map[byte]int{}
	left, ans := 0, 0
	for right := 0; right < len(s); right++ {
		cnt[s[right]]++
		for cnt[s[right]] > 1 {
			cnt[s[left]]--
			left++
		}
		if size := right - left + 1; size > ans {
			ans = size
		}
	}
	return ans
}
```

## Rust 参考骨架

```rust
pub fn min_sub_array_len(target: i32, nums: Vec<i32>) -> i32 {
    let (mut left, mut sum) = (0usize, 0);
    let mut ans = nums.len() + 1;
    for right in 0..nums.len() {
        sum += nums[right];
        while sum >= target {
            ans = ans.min(right - left + 1);
            sum -= nums[left];
            left += 1;
        }
    }
    if ans == nums.len() + 1 { 0 } else { ans as i32 }
}
```

## 为什么这样写

暴力做法会枚举所有起点和终点，再检查每个窗口是否合法。滑动窗口把这个过程拆成两件事：右端只负责扩张，左端只在窗口不符合当前目标时收缩。

每个元素最多被右指针加入一次、被左指针移出一次，所以扫描次数是线性的。计数表的作用，是让“加入一个元素”和“移出一个元素”都不需要重新遍历窗口。

要特别区分两类题：

- 最长合法窗口：收缩直到合法，然后更新最大值。
- 最短满足窗口：一旦满足条件，就在 `while` 中更新答案并尝试继续缩短。

## 复杂度

- 时间复杂度：$O(n)$，左右指针各自最多走完整个数组或字符串。
- 空间复杂度：$O(k)$，`k` 是字符集大小或窗口中不同元素的数量；只维护和时是 $O(1)$。

## 易错点

- 收缩时只移动了 `left`，忘记把 `s[left]` 从计数表里减掉。
- 最短窗口题在满足条件后没有立刻更新答案，错过当前窗口。
- 固定长度窗口题忘记限制窗口长度，导致计数表包含了过期字符。
- 用字符计数判断匹配时，每次比较整张表可以通过，但要知道字符集变大后会拖慢常数。
