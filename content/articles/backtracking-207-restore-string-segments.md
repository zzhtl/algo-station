# 字符串分段回溯：回溯精讲

## 一句话总结

字符串分段的选择不是字符，而是下一段的结束位置。

## 识别信号

复原 IP、累加数、斐波那契拆分、回文分割。

对「字符串分段回溯」来说，识别信号不是关键词本身，而是它们都在要求你在搜索树里明确每层负责哪一个选择，并保证选择、递归、撤销三步成对出现。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 选择与撤销锚点

- 本篇围绕「字符串分段回溯」训练：字符串分段的选择不是字符，而是下一段的结束位置。
- 路径里保存的是当前层已经确认的选择，离开这一层时必须恢复到进入前的状态。
- 识别信号里的“复原 IP、累加数、斐波那契拆分、回文分割”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #93、#306、#842、#131 时，复盘「选择列表、剪枝条件、撤销位置」是否还能用同一套解释串起来。

## 推导步骤

1. 从 start 枚举段长。
2. 检查前导零、范围和关系约束。
3. 加入段后递归。
4. 段数超限剪枝。

## 进入递归前的边界清单

- 空输入、重复元素、无解、刚好满足约束、越界剪枝都要测。
- 对照「字符串分段回溯」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #93、#306、#842、#131 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 前导零允许了多位数。
- 整数溢出。
- 剩余长度不可能仍继续搜索。
