# 数位 DP 的 tight 与 started：动态规划精讲

## 一句话总结

只要前缀小于上界，后面位就可以自由选择；前导零需要单独建模。

## 识别信号

数字 1 个数、不含连续 1、可用数字组成数、重复数字。

对「数位 DP 的 tight 与 started」来说，识别信号不是关键词本身，而是它们都在要求你把重复子问题压成状态，并保证每一次转移只从已经定义清楚的旧状态来。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 状态转移锚点

- 本篇围绕「数位 DP 的 tight 与 started」训练：只要前缀小于上界，后面位就可以自由选择；前导零需要单独建模。
- 状态含义要先于代码确定；如果一句话说不清 dp[i] 表示什么，转移大概率会写散。
- 识别信号里的“数字 1 个数、不含连续 1、可用数字组成数、重复数字”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #233、#600、#902、#1012 时，复盘「状态定义、转移来源、遍历顺序」是否还能用同一套解释串起来。

## 推导步骤

1. 把数字转成数组。
2. 递归位置、tight、started 和其他状态。
3. 记忆化非 tight 状态。

## 写转移前的边界清单

- 初始状态、不可能状态、容量或下标为 0、答案取 max 还是求和要分清。
- 对照「数位 DP 的 tight 与 started」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #233、#600、#902、#1012 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 把前导零计入数字。
- tight 状态也盲目缓存。
- 上界本身是否包含没处理。
