# 数据流中位数双堆：设计与数据结构训练题解

作为设计题，数据流中位数要先定复杂度目标：不断 `AddNum`，随时 `FindMedian`。排序数组插入慢，单堆拿不到中间位置，所以使用两个堆。

## 结构不变量

- 左堆 `small` 是大根堆，保存较小的一半。
- 右堆 `large` 是小根堆，保存较大的一半。
- `small` 的大小等于 `large`，或多一个。
- `small` 堆顶不大于 `large` 堆顶。

查询时：

- 奇数个元素：返回 `small` 堆顶。
- 偶数个元素：返回两个堆顶平均值。

## 操作流程

插入一个数时：

1. 根据 `small` 堆顶决定放左堆还是右堆。
2. 如果左堆太大，把左堆顶搬到右堆。
3. 如果右堆更大，把右堆顶搬到左堆。

搬运边界值不会破坏左右顺序。

## Go 实现要点

Go 的 `container/heap` 需要自定义 `Less`。可以写两个堆类型，也可以写一个带比较函数的堆。训练时建议先写清楚两个堆的语义，再复用堆代码。

```go
type IntHeap struct {
	data []int
	less func(a, b int) bool
}
```

`small` 的 `less` 是 `a > b`，`large` 的 `less` 是 `a < b`。其余 `Push/Pop` 方法相同。

## 易错点

- 只平衡大小不够，还要保证左右顺序。
- 约定左堆多一个元素后，奇数查询更简单。
- 偶数平均值注意整数溢出和浮点转换。

## 复杂度

插入 `O(log n)`，查询 `O(1)`，空间 `O(n)`。
