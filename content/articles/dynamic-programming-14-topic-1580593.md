# 子数组最大和在线转移：动态规划训练题解

最大子数组和要求子数组连续，所以状态必须强调“以当前位置结尾”。如果只定义成前 i 个数的最大和，转移会丢失连续性。

一句话记法：**前面贡献为负就断开，从当前元素重新开始。**

## 状态定义

```text
dp[i] = 以 nums[i] 结尾的最大子数组和
```

转移：

```text
dp[i] = max(nums[i], dp[i-1] + nums[i])
```

答案是所有 `dp[i]` 的最大值。

## Go 参考实现

```go
func maxSubArray(nums []int) int {
	cur, ans := nums[0], nums[0]
	for i := 1; i < len(nums); i++ {
		if cur < 0 {
			cur = nums[i]
		} else {
			cur += nums[i]
		}
		if cur > ans {
			ans = cur
		}
	}
	return ans
}
```

## 为什么这样写

以 `i` 结尾的连续子数组只有两种：

- 只包含 `nums[i]`，从当前重新开始。
- 接在以 `i-1` 结尾的最大子数组后面。

如果前一段最大和是负数，接上它只会让当前更小，所以重新开始更优。

## 环形数组变体

#918 环形最大子数组和有两种情况：

- 不跨首尾：普通最大子数组和。
- 跨首尾：总和减去中间一段最小子数组和。

如果全是负数，不能用 `total - minSubarray` 得到空数组，要单独处理。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(1)$。

## 易错点

- 状态没写“以 i 结尾”，导致转移不闭合。
- 全负数组时把答案初始化为 0。
- 环形版本没有处理全负情况。
- 把子序列和子数组混淆，子数组必须连续。
