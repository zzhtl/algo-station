# 字符串双序列 DP：动态规划精讲

## 一句话总结

末尾字符是否相同决定能否从左上角继承，否则从删除、插入、跳过中选择。

## 识别信号

编辑距离、最长公共子序列、不同子序列、删除操作。

对「字符串双序列 DP」来说，识别信号不是关键词本身，而是它们都在要求你把重复子问题压成状态，并保证每一次转移只从已经定义清楚的旧状态来。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 状态转移锚点

- 本篇围绕「字符串双序列 DP」训练：末尾字符是否相同决定能否从左上角继承，否则从删除、插入、跳过中选择。
- 状态含义要先于代码确定；如果一句话说不清 dp[i] 表示什么，转移大概率会写散。
- 识别信号里的“编辑距离、最长公共子序列、不同子序列、删除操作”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #72、#1143、#115、#583 时，复盘「状态定义、转移来源、遍历顺序」是否还能用同一套解释串起来。

## 推导步骤

1. 定义两个前缀状态。
2. 初始化空串行列。
3. 枚举 i/j。
4. 按末尾字符关系转移。

## 写转移前的边界清单

- 初始状态、不可能状态、容量或下标为 0、答案取 max 还是求和要分清。
- 对照「字符串双序列 DP」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #72、#1143、#115、#583 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- i/j 和字符下标差一位。
- 空串初始化漏掉。
- 子串和子序列混淆。
