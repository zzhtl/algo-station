# 规范化 key 合并等价对象：哈希与前缀精讲

## 一句话总结

只要两个对象在题目意义下等价，就应该映射到同一个稳定 key。

## 识别信号

字母异位词、移位字符串、共线点、岛屿形状。

对「规范化 key 合并等价对象」来说，识别信号不是关键词本身，而是它们都在要求你把重复查询改成一次记录，让“以前见过什么”和“当前还缺什么”可以 O(1) 对上。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 计数与映射锚点

- 本篇围绕「规范化 key 合并等价对象」训练：只要两个对象在题目意义下等价，就应该映射到同一个稳定 key。
- 哈希表里保存的必须是可复用的证据，而不是碰巧能过样例的中间值。
- 识别信号里的“字母异位词、移位字符串、共线点、岛屿形状”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #49、#249、#149、#694 时，复盘「键的定义、值的含义、首次或最新出现策略」是否还能用同一套解释串起来。

## 推导步骤

1. 分析等价条件。
2. 选择排序串、计数向量、约分斜率或相对坐标作为 key。
3. 按 key 聚合。

## 建表前的边界清单

- 重复值、负数、前缀为 0、同一个键多次出现要单独验。
- 对照「规范化 key 合并等价对象」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #49、#249、#149、#694 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- key 不唯一。
- 浮点斜率精度问题。
- 坐标形状没有归一到起点。
