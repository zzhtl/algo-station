# 反向思考最大化剩余：贪心训练题解

贪心不一定是直接选答案。有些题反向想更清楚：不是问“怎么选最好”，而是问“怎么删损失最小”或“怎么切不影响后面”。

一句话记法：**正向难证明时，换成保留余量或排除不可能。**

## 例：划分字母区间

```go
func partitionLabels(s string) []int {
	last := [26]int{}
	for i := range s {
		last[s[i]-'a'] = i
	}
	ans := []int{}
	start, end := 0, 0
	for i := range s {
		if last[s[i]-'a'] > end {
			end = last[s[i]-'a']
		}
		if i == end {
			ans = append(ans, end-start+1)
			start = i + 1
		}
	}
	return ans
}
```

## 为什么这样写

一个片段里出现的所有字符，最后一次出现位置都必须落在这个片段内。扫描时维护当前片段必须覆盖到的最远 `end`；当 `i == end`，说明当前片段已经闭合，可以立刻切开。

越早切开，后面剩余空间越大，也不会破坏当前片段合法性。

## 更多例子

- 柠檬水找零：优先保留更有用的 5 元零钱，给 20 找零时尽量用 `10+5`。
- 摆动序列：只保留方向变化的峰谷，中间同向点删除不影响最长长度。
- 区间删除：保留右端更小的区间，等价于删除更挡路的区间。

## 复杂度

- 通常是 $O(n)$ 或排序后 $O(n \log n)$。
- 空间复杂度看是否需要计数表或排序。

## 易错点

- 只看当前字符，不看它最后出现位置。
- 柠檬水找零给 20 时优先用三个 5，导致 10 元无法使用。
- 摆动序列把相等差值当成方向变化。
- 反向思考后没有回到原题目标验证。
