# 组合问题的 start 下标：回溯精讲

## 一句话总结

用 start 保证每个集合只生成一次，避免 `[1,2]` 和 `[2,1]` 重复。

## 识别信号

组合、子集、含重复子集、组合总和 III。

对「组合问题的 start 下标」来说，识别信号不是关键词本身，而是它们都在要求你在搜索树里明确每层负责哪一个选择，并保证选择、递归、撤销三步成对出现。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 选择与撤销锚点

- 本篇围绕「组合问题的 start 下标」训练：用 start 保证每个集合只生成一次，避免 `[1,2]` 和 `[2,1]` 重复。
- 路径里保存的是当前层已经确认的选择，离开这一层时必须恢复到进入前的状态。
- 识别信号里的“组合、子集、含重复子集、组合总和 III”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #77、#78、#90、#216 时，复盘「选择列表、剪枝条件、撤销位置」是否还能用同一套解释串起来。

## 推导步骤

1. 递归参数带 start。
2. 循环从 start 开始。
3. 选择当前数后递归 `i+1`。
4. 需要重复使用时递归 `i`。

## 进入递归前的边界清单

- 空输入、重复元素、无解、刚好满足约束、越界剪枝都要测。
- 对照「组合问题的 start 下标」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #77、#78、#90、#216 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 组合和排列混淆。
- 重复元素没有同层去重。
- 剪枝边界过强漏解。
