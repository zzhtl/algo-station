# 差分数组批量更新：数组与字符串训练题解

差分数组适合“很多次区间加减，最后才关心每个位置结果”的题。它的核心很简单：要让 `[l, r]` 整段加 `v`，不必逐个位置加，只需要在差分数组里做两次端点修改。

闭区间写法是：`diff[l] += v`，如果 `r + 1 < n`，再执行 `diff[r + 1] -= v`。最后对 `diff` 做一次前缀和，就能还原每个位置的真实值。

## 适用场景

当更新次数很多、每次更新是一段连续区间，并且中间不需要频繁查询单点值时，优先考虑差分。

- 航班预订、拼车：每个订单影响一段站点或航班。
- 区间覆盖判断：统计每个点被覆盖了几次。
- 批量加减后判断是否全满足限制。
- 二维矩阵的矩形加法，也可以扩展成二维差分。

如果每次更新后都要立刻查询任意区间结果，差分未必够用，可能需要线段树或树状数组。

## 图解思路

```mermaid
flowchart LR
  A[区间 l..r 加 v] --> B[diff[l] += v]
  B --> C{r + 1 是否越界?}
  C -->|否| D[diff[r + 1] -= v]
  C -->|是| E[跳过右端回退]
  D --> F[所有更新结束]
  E --> F
  F --> G[前缀和还原数组]
```

差分数组记录的不是当前位置的真实值，而是“从这个位置开始，增量发生了什么变化”：

- `diff[l] += v` 表示从 `l` 开始多加 `v`。
- `diff[r + 1] -= v` 表示从 `r + 1` 开始取消这次影响。
- 前缀和把所有仍然生效的增量累加起来，得到当前位置结果。

## 手写步骤

1. 明确题目区间是闭区间 `[l, r]` 还是半开区间 `[l, r)`。
2. 建立长度为 `n` 或 `n + 1` 的差分数组。
3. 对每个区间更新左右端点。
4. 扫描差分数组做前缀和，还原真实值或边还原边判断条件。
5. 检查输入下标是否从 1 开始，需要转换成 0 基。

## Go 参考骨架

```go
func corpFlightBookings(bookings [][]int, n int) []int {
	diff := make([]int, n+1)
	for _, b := range bookings {
		left, right, seats := b[0]-1, b[1]-1, b[2]
		diff[left] += seats
		diff[right+1] -= seats
	}
	ans := make([]int, n)
	cur := 0
	for i := 0; i < n; i++ {
		cur += diff[i]
		ans[i] = cur
	}
	return ans
}
```

## Rust 参考骨架

```rust
pub fn car_pooling(trips: Vec<Vec<i32>>, capacity: i32) -> bool {
    let mut diff = vec![0; 1001];
    for trip in trips {
        let passengers = trip[0];
        let from = trip[1] as usize;
        let to = trip[2] as usize;
        diff[from] += passengers;
        diff[to] -= passengers;
    }

    let mut cur = 0;
    for delta in diff {
        cur += delta;
        if cur > capacity {
            return false;
		}
	}
    true
}
```

## 为什么这样写

如果有 `m` 次区间更新，每次都逐个位置修改，复杂度可能是 $O(mn)$。差分把每次区间更新压成 $O(1)$，最后只用一次 $O(n)$ 前缀扫描还原。

以 #1109 航班预订为例，一条预订 `[first, last, seats]` 会让从 `first` 到 `last` 的航班都增加座位数。差分只在 `first` 处开始增加，在 `last + 1` 处停止增加。前缀扫描时，这段影响自然会覆盖中间所有航班。

以 #1094 拼车为例，它是半开区间：乘客在 `from` 上车，在 `to` 下车，所以影响的是 `[from, to)`，写成 `diff[from] += passengers` 和 `diff[to] -= passengers`。

## 复杂度

- 时间复杂度：$O(n + m)$，`m` 是区间更新次数。
- 空间复杂度：$O(n)$，差分数组长度与坐标范围或数组长度有关。

## 易错点

- 闭区间 `[l, r]` 忘记在 `r + 1` 处减回去。
- 输入是 1 基下标，直接当 0 基使用导致整体偏移。
- 半开区间 `[from, to)` 被误写成闭区间，导致下车站仍然占座。
- 差分数组长度开成 `n` 时，写 `r + 1` 需要边界判断；开成 `n + 1` 可以减少分支。
