# 矩阵坐标变换与层次遍历：数组与字符串精讲

## 一句话总结

坐标变换要先确定遍历顺序，再把边界收缩、方向切换和越界回退拆开处理。

## 识别信号

旋转矩阵、螺旋矩阵、对角线遍历、螺旋坐标。

对「矩阵坐标变换与层次遍历」来说，识别信号不是关键词本身，而是它们都在要求你在一段连续结构里保留足够信息，让窗口、指针或扫描线每推进一步都能解释答案为什么没有被漏掉。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 指针与区间锚点

- 本篇围绕「矩阵坐标变换与层次遍历」训练：坐标变换要先确定遍历顺序，再把边界收缩、方向切换和越界回退拆开处理。
- 数组题的核心不是多写循环，而是让每个下标只承担一种角色：左边界、右边界、候选值或答案落点。
- 识别信号里的“旋转矩阵、螺旋矩阵、对角线遍历、螺旋坐标”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。

## 推导步骤

1. 定义方向数组或层边界。
2. 每步尝试移动。
3. 越界或访问过就转向。
4. 层遍历时收缩上下左右边界。

## 动手前的边界清单

- 空数组、单元素、全相等、严格递增或递减都要先手算一遍。
- 对照「矩阵坐标变换与层次遍历」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #48、#54、#59、#885、#498 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 方向切换后没有回退。
- 单行单列重复访问。
- 层边界收缩顺序错误。
