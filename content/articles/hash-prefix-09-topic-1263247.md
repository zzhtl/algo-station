# 状态压缩成字符串键：哈希与前缀训练题解

哈希表的关键是 key。复杂状态不能随便拼，要保证同一状态得到同一 key，不同状态尽量不冲突。

## 常见 key 设计

- 小写字母计数：`[26]int`。
- 坐标：结构体 `{r, c}` 或 `r*n+c`。
- 多字段：结构体或带分隔符字符串。
- 集合状态：bitmask。

## Go 参考：数独小方格 key

```go
type Cell struct {
	Box int
	Num byte
}

func boxID(r, c int) int {
	return (r/3)*3 + c/3
}
```

对于数独，也可以不用 map，直接用 `[9][10]bool` 表示行、列、宫。

## 为什么这样写

错误 key 会制造两类问题：

- 同一状态得到不同 key，缓存失效。
- 不同状态得到同一 key，答案错误。

比如坐标直接拼 `"1"+"23"` 和 `"12"+"3"` 都是 `"123"`，必须加分隔符或用整数编码。

## 易错点

- 切片不能直接作为 Go map key。
- 字符串拼接没有分隔符导致冲突。
- bitmask 位数不够。
- key 中漏掉一个状态维度。
