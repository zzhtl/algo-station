# 局部最优配饼干：贪心训练题解

饼干分配的目标是满足尽量多孩子。排序后，用当前最小可用饼干去满足当前最小胃口孩子，是最不浪费的选择。

一句话记法：**小胃口配小饼干，能满足就成交，不能满足就换大饼干。**

## Go 参考实现

```go
func findContentChildren(g []int, s []int) int {
	sort.Ints(g)
	sort.Ints(s)
	i, j, ans := 0, 0, 0
	for i < len(g) && j < len(s) {
		if s[j] >= g[i] {
			ans++
			i++
			j++
		} else {
			j++
		}
	}
	return ans
}
```

## 为什么这样写

如果最小饼干能满足最小胃口孩子，把它给别人不会更优；如果最小饼干连最小胃口都满足不了，它也满足不了任何剩余孩子，只能丢弃。

这就是排序双指针贪心的交换依据。

## 复杂度

- 时间复杂度：$O(n \log n + m \log m)$。
- 空间复杂度：取决于排序实现。

## 易错点

- 不排序直接匹配。
- 大饼干优先给小胃口但没有证明；排序双指针更稳。
- 饼干不够时推进了孩子指针。
- 统计的是满足孩子数，不是用掉饼干数。
