# 双指针相交节点：链表训练题解

相交链表的难点是两条链长度不同。让长链先走差值可以做，但更简洁的是双指针换路：A 走完去 B，B 走完去 A。

一句话记法：**你走我的路，我走你的路，总路程相等后自然对齐。**

## 适用场景

- 判断两条单链表是否在某个节点相交。
- 相交定义是节点引用相同，不是节点值相等。
- 链表无环。

如果链表可能有环，相交问题要先分类讨论环入口。

## 图解思路

```text
A: a1 -> a2 -> c1 -> c2
B: b1 -> b2 -> b3 -> c1 -> c2

pA 走 A 再走 B
pB 走 B 再走 A
两者都会在 c1 对齐
```

如果没有交点，两者最终都会走到 `nil`，循环也会结束。

## 不变量

- `pA` 和 `pB` 每轮各走一步。
- `pA` 到 A 尾后切到 B 头。
- `pB` 到 B 尾后切到 A 头。
- 两个指针走过的总长度最终相等。

## 手写步骤

1. 初始化 `pA := headA, pB := headB`。
2. 当 `pA != pB` 时循环。
3. `pA == nil` 就跳到 `headB`，否则走下一步。
4. `pB == nil` 就跳到 `headA`，否则走下一步。
5. 返回 `pA`，可能是交点，也可能是 nil。

## Go 参考实现

```go
func getIntersectionNode(headA, headB *ListNode) *ListNode {
	pA, pB := headA, headB
	for pA != pB {
		if pA == nil {
			pA = headB
		} else {
			pA = pA.Next
		}
		if pB == nil {
			pB = headA
		} else {
			pB = pB.Next
		}
	}
	return pA
}
```

## 为什么这样写

设 A 独有长度为 `a`，B 独有长度为 `b`，公共尾部长度为 `c`。指针换路后：

- `pA` 走的长度是 `a + c + b`。
- `pB` 走的长度是 `b + c + a`。

两者在进入公共尾部前走过的长度被补齐，所以会在第一个公共节点相遇。若没有公共尾部，两个指针都会在走完 `a+b` 后变成 nil。

## 复杂度

- 时间复杂度：$O(m+n)$。
- 空间复杂度：$O(1)$。

## 易错点

- 用节点值比较，遇到相同值但不同节点会误判。
- 指针到 nil 后没有切换到另一条链。
- 只让长链先走但长度计算错。
- 有环链表直接套这个方法，可能无限循环。
