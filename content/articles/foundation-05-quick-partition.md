# 快速排序划分：让 pivot 一次到位

快速排序的关键不是递归，而是 `partition`：选择一个基准值 `pivot`，重新排列区间，使它左边不大于它、右边大于它，然后只递归处理两侧。

## Lomuto 分区不变量

选择末尾元素作为 pivot，扫描指针 `j` 从左向右，边界 `i` 指向下一个应该放“小元素”的位置：

- `[l, i)` 中元素都不大于 pivot；
- `[i, j)` 中元素都大于 pivot；
- `[j, r)` 尚未检查；
- `a[r]` 是 pivot。

遇到 `a[j] <= pivot` 时，交换 `a[i]` 与 `a[j]` 并增加 `i`。扫描结束后交换 `a[i]` 与 pivot，`i` 就是基准的最终位置。

```go
func partition(a []int, l, r int) int {
    pivot := a[r]
    i := l
    for j := l; j < r; j++ {
        if a[j] <= pivot {
            a[i], a[j] = a[j], a[i]
            i++
        }
    }
    a[i], a[r] = a[r], a[i]
    return i
}
```

## 复杂度与退化

每次划分扫描当前区间一次。若 pivot 大致把数组平分，递归高度为 $O(\log n)$，总时间 $O(n\log n)$；若每次都选到最小或最大值，递归高度变成 $n$，最坏时间 $O(n^2)$，栈也会变深。

常见缓解方法是随机选择 pivot，或使用三数取中。重复值很多时，三向切分把区间分成 `< pivot`、`== pivot`、`> pivot`，避免相等元素反复进入递归。

## 与归并排序对比

快速排序平均性能和缓存局部性很好，数组上通常原地工作，但不稳定且有最坏情况。归并排序时间上界稳定、可以稳定排序，但数组版本需要辅助空间。

## 易错点

- 扫描范围包含 pivot 自己。
- 忘记最后让 pivot 与 `a[i]` 交换。
- 递归仍包含 pivot 位置，区间不缩小。
- 对重复值使用不合适的比较条件，造成极不平衡切分。

记住 `i` 的含义比背交换顺序更可靠：它始终是“小元素区”右侧的第一个空位。
