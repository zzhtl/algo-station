# 原地交换与稳定写入：数组与字符串训练题解

原地修改题最容易混淆两种手法：交换和稳定写入。它们都能做到 $O(1)$ 额外空间，但适用条件不同。

先问一个问题：**题目是否要求保留元素的相对顺序**？如果要求稳定顺序，用写指针从左到右覆盖；如果不要求稳定顺序，交换可以更快地把元素放到目标区域。

## 适用场景

这类题通常明确要求“原地”，或者输入数组本身就是输出容器。

- 稳定写入：移动零、删除元素、合并时从后往前写。
- 原地交换：反转字符串、旋转数组、颜色分类。
- 双端填充：有序数组平方，较大值从结果末尾写入。
- 三路划分：把小于、等于、大于某个值的元素划成三段。

如果输出顺序和输入顺序有关，交换前要谨慎；交换通常会改变未处理元素的位置。

## 图解思路

```mermaid
flowchart LR
  A{是否要求稳定顺序?} -->|是| B[read 扫描]
  B --> C[符合条件就写入 nums[write]]
  C --> D[write 后移]
  A -->|否| E[维护目标区间边界]
  E --> F[通过 swap 把元素放进对应区域]
  F --> G[更新区间边界]
```

写之前先把数组分成几段：

- 稳定写入：`[0, write)` 是已经完成的结果前缀，`[read, n)` 还没读。
- 交换分区：`[0, low)`、`[low, mid)`、`(high, n)` 分别代表已经归位的区域。
- 从后写入：`write` 指向最终结果的空位，避免覆盖前面还没读取的元素。

## 手写步骤

1. 判断题目是否要求稳定顺序。
2. 如果稳定，使用 `read/write`，只把保留元素写到前缀。
3. 如果不稳定，定义每个区间的含义，再用交换维护区间。
4. 如果从前写会覆盖未读元素，就改成从后往前写。
5. 每次移动指针前，确认被跳过或被交换的元素已经处理完。

## Go 参考骨架

```go
func sortColors(nums []int) {
	low, mid, high := 0, 0, len(nums)-1
	for mid <= high {
		switch nums[mid] {
		case 0:
			nums[low], nums[mid] = nums[mid], nums[low]
			low++
			mid++
		case 1:
			mid++
		default:
			nums[mid], nums[high] = nums[high], nums[mid]
			high--
		}
	}
}
```

## Rust 参考骨架

```rust
pub fn sorted_squares(nums: Vec<i32>) -> Vec<i32> {
    let n = nums.len();
    let (mut left, mut right) = (0usize, n - 1);
    let mut ans = vec![0; n];

    for write in (0..n).rev() {
        if nums[left].abs() > nums[right].abs() {
            ans[write] = nums[left] * nums[left];
            left += 1;
        } else {
            ans[write] = nums[right] * nums[right];
            if right == 0 {
                break;
            }
            right -= 1;
        }
    }
    ans
}
```

## 为什么这样写

以 #75 颜色分类为例，它不要求稳定顺序，所以可以维护三段区域：左边全是 `0`，右边全是 `2`，中间扫描未知元素。遇到 `0` 就换到左侧，遇到 `2` 就换到右侧，遇到 `1` 直接前进。

以 #88 合并两个有序数组为例，情况则相反：如果从前往后写，会覆盖 `nums1` 里还没读的元素。因此应该从两个数组末尾比较，把较大的数写到 `nums1` 的末尾。

同样是原地，关键在于“写入会不会破坏未来还要读的数据”。

## 复杂度

- 时间复杂度：多数原地扫描题是 $O(n)$。
- 空间复杂度：交换和稳定覆盖通常是 $O(1)$；如果题目要求返回新数组，则输出数组不算额外优化空间。

## 易错点

- #75 中把 `2` 换到右侧后不能立刻 `mid++`，因为换回来的元素还没检查。
- #88 从前往后合并会覆盖 `nums1` 中未处理的值。
- 稳定写入时先移动 `write` 再赋值，会错过第一个写入位置。
- Rust 使用 `usize` 做右指针时，要避免减到 0 后继续下溢。

## 练习顺序

建议按这个顺序刷：#344, #283, #88, #977, #75, #189。

先用反转字符串和移动零区分交换、稳定写入，再用合并数组练从后写，最后做颜色分类和旋转数组练多段区间不变量。
