# 双端队列维护窗口最大值：栈与队列训练题解

滑动窗口最大值不能每个窗口都重新扫描。单调队列维护一组候选最大值，下标从队头到队尾递增，值从队头到队尾递减。

一句话记法：**队头是最大值，队尾负责淘汰更小候选。**

## 适用场景

- 固定长度滑动窗口最大值/最小值。
- 需要每次窗口移动后 $O(1)$ 取最值。
- 元素按时间顺序进入，过期元素按下标离开。

最大值用递减队列，最小值用递增队列。

## 图解思路

```mermaid
flowchart LR
  A[新下标 i 进入] --> B[队头过期则弹出]
  B --> C[队尾比 nums[i] 小则弹出]
  C --> D[i 入队]
  D --> E[窗口形成后 nums[队头] 是最大值]
```

队列里存下标，不存值，因为要判断是否过期。

## 不变量

- 队列中的下标递增。
- 队列中的值递减。
- 队头下标始终在当前窗口内。
- 队头对应值就是当前窗口最大值。

## Go 参考实现

```go
func maxSlidingWindow(nums []int, k int) []int {
	q := []int{}
	ans := []int{}
	for i, x := range nums {
		if len(q) > 0 && q[0] <= i-k {
			q = q[1:]
		}
		for len(q) > 0 && nums[q[len(q)-1]] <= x {
			q = q[:len(q)-1]
		}
		q = append(q, i)
		if i >= k-1 {
			ans = append(ans, nums[q[0]])
		}
	}
	return ans
}
```

## 为什么这样写

如果队尾元素小于等于新元素，那么它既比新元素更早过期，又不比新元素大，未来不可能成为最大值，可以直接删掉。

队头过期必须优先处理。窗口右端到 `i` 时，合法下标范围是 `[i-k+1, i]`，所以 `q[0] <= i-k` 的下标已经离开窗口。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(k)$，队列最多保留窗口内候选。

## 易错点

- 队列存值，无法判断过期。
- 过期条件写成 `< i-k`，导致旧下标多留一轮。
- 维护最大值时队尾弹出条件写反。
- 窗口还没形成就提前记录答案。
