# 网格 DFS 染色：搜索与图论训练题解

网格 DFS 适合处理连通块。每个格子是一个节点，上下左右相邻是边。染色的意思是：访问过的格子立刻标记，避免重复进入。

一句话记法：**遇到一块新陆地就 DFS，把整座岛染掉。**

## 适用场景

- 岛屿数量。
- 岛屿最大面积。
- #130 这类边界连通区域题。
- 只关心连通性，不关心最短步数。

如果题目问最短距离，通常用 BFS；如果有权重，考虑最短路算法。

## Go 参考实现：岛屿数量

```go
func numIslands(grid [][]byte) int {
	m, n := len(grid), len(grid[0])
	var dfs func(int, int)
	dfs = func(r, c int) {
		if r < 0 || r >= m || c < 0 || c >= n || grid[r][c] != '1' {
			return
		}
		grid[r][c] = '0'
		dfs(r+1, c)
		dfs(r-1, c)
		dfs(r, c+1)
		dfs(r, c-1)
	}

	ans := 0
	for r := 0; r < m; r++ {
		for c := 0; c < n; c++ {
			if grid[r][c] == '1' {
				ans++
				dfs(r, c)
			}
		}
	}
	return ans
}
```

## 为什么这样写

外层循环负责发现新的连通块，DFS 负责把这个连通块全部访问。只要 DFS 中把访问过的陆地改成水，外层就不会重复计数。

染色可以原地修改网格，也可以用 `visited` 数组。原地修改省空间，但会改变输入。

## 复杂度

- 时间复杂度：$O(mn)$。
- 空间复杂度：递归栈最坏 $O(mn)$。

## 易错点

- 访问后不标记，导致无限递归。
- 边界判断写在访问数组之后，先越界再取值。
- 八方向和四方向混用。
- #130 从内部出发难以判断是否贴边；应从边界反向标记。
