# 单调递减栈找下一个更大：栈与队列训练题解

“下一个更大元素”是单调栈最标准的模型。扫描到当前元素时，它可能是栈中若干旧元素的第一个更大值。

一句话记法：**栈里存还没等到更大值的下标；当前值更大，就替它们结算。**

## 适用场景

- 下一个更大元素。
- 每日温度：下一个更高温度的距离。
- 循环数组的下一个更大。
- 股票跨度的反向模型。

想找“更大”，栈就维护递减；想找“更小”，反过来维护递增。

## 图解思路

```mermaid
flowchart LR
  A[当前 nums[i]] --> B{"nums[i] > nums[栈顶]?"}
  B -->|是| C[栈顶找到答案并弹出]
  C --> B
  B -->|否| D[i 入栈等待未来更大值]
```

被弹出的下标，它们的答案就是当前 `i` 或当前值。

## 不变量

- 栈内下标对应的值从底到顶递减。
- 栈里每个下标都还没找到右侧第一个更大值。
- 当当前值弹出栈顶时，当前值就是那个栈顶的第一个更大值。
- 每个下标最多入栈一次、出栈一次。

## Go 参考实现：每日温度

```go
func dailyTemperatures(temperatures []int) []int {
	ans := make([]int, len(temperatures))
	st := []int{}
	for i, t := range temperatures {
		for len(st) > 0 && temperatures[st[len(st)-1]] < t {
			j := st[len(st)-1]
			st = st[:len(st)-1]
			ans[j] = i - j
		}
		st = append(st, i)
	}
	return ans
}
```

## 循环数组怎么处理

循环数组可以扫描两轮，第二轮只用来结算，不再压入新下标：

```go
func nextGreaterElements(nums []int) []int {
	n := len(nums)
	ans := make([]int, n)
	for i := range ans {
		ans[i] = -1
	}
	st := []int{}
	for i := 0; i < 2*n; i++ {
		cur := nums[i%n]
		for len(st) > 0 && nums[st[len(st)-1]] < cur {
			j := st[len(st)-1]
			st = st[:len(st)-1]
			ans[j] = cur
		}
		if i < n {
			st = append(st, i)
		}
	}
	return ans
}
```

## 为什么这样写

栈顶元素如果小于当前值，它的下一个更大值一定是当前值。因为它从入栈到现在都没有遇到更大的值，否则早就被弹出了。

栈保持递减，是为了让当前值能一次性结算一批较小元素。剩下没弹出的元素都大于等于当前值，当前值无法成为它们的答案。

## 复杂度

- 时间复杂度：$O(n)$，循环数组是 $O(2n)$ 仍为 $O(n)$。
- 空间复杂度：$O(n)$。

## 易错点

- 栈里存值，重复元素时无法回填正确下标。
- 弹栈条件写成 `<=`，导致“更大”和“大于等于”混淆。
- 循环数组第二轮继续压栈，产生重复下标。
- 每日温度答案要填距离 `i-j`，不是温度值。
