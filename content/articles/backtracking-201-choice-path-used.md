# 选择路径和 used 集合：回溯精讲

## 一句话总结

排列关注位置，每个位置都要从剩余候选中选一个。

## 识别信号

全排列、含重复全排列、字母大小写排列、活字印刷。

对「选择路径和 used 集合」来说，识别信号不是关键词本身，而是它们都在要求你在搜索树里明确每层负责哪一个选择，并保证选择、递归、撤销三步成对出现。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 选择与撤销锚点

- 本篇围绕「选择路径和 used 集合」训练：排列关注位置，每个位置都要从剩余候选中选一个。
- 路径里保存的是当前层已经确认的选择，离开这一层时必须恢复到进入前的状态。
- 识别信号里的“全排列、含重复全排列、字母大小写排列、活字印刷”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #46、#47、#784、#1079 时，复盘「选择列表、剪枝条件、撤销位置」是否还能用同一套解释串起来。

## 推导步骤

1. 递归层数等于 path 长度。
2. 遍历候选。
3. 跳过已使用。
4. 选择后递归，返回时撤销。

## 进入递归前的边界清单

- 空输入、重复元素、无解、刚好满足约束、越界剪枝都要测。
- 对照「选择路径和 used 集合」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #46、#47、#784、#1079 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 去重条件没和排序配合。
- 撤销 used 或 path 漏掉。
- 组合题误写成排列。
