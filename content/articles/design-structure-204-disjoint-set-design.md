# 并查集接口设计：设计与数据结构精讲

## 一句话总结

集合合并后，代表元承载整个集合的统计信息。

## 识别信号

省份数量、账户合并、等式方程、小交换字典序最小。

对「并查集接口设计」来说，识别信号不是关键词本身，而是它们都在要求你把每个操作的时间复杂度要求翻译成内部结构之间的同步关系。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 接口语义锚点

- 本篇围绕「并查集接口设计」训练：集合合并后，代表元承载整个集合的统计信息。
- 外部接口看到的是一个结构，内部多个容器必须在每次操作后保持同一份事实。
- 识别信号里的“省份数量、账户合并、等式方程、小交换字典序最小”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #547、#721、#990、#1202 时，复盘「接口承诺、内部同步、复杂度来源」是否还能用同一套解释串起来。

## 推导步骤

1. 初始化 parent/size。
2. find 路径压缩。
3. union 按大小合并。
4. 需要时维护集合成员。

## 定结构前的边界清单

- 重复插入、删除不存在、容量边界、查询空结构、时间戳相同都要覆盖。
- 对照「并查集接口设计」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #547、#721、#990、#1202 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 只改一个节点 parent 没合并根。
- 扩展信息没有随根迁移。
- 字符串 key 没映射成 id。
