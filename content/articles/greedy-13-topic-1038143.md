# 单调栈做贪心删除：贪心训练题解

删除数字让结果最小，本质是让高位尽量小。当前数字比栈顶小，且还有删除次数，就应该删掉栈顶。

一句话记法：**能删就删前面更大的数，让小数尽早出现。**

## Go 参考实现

```go
func removeKdigits(num string, k int) string {
	st := []byte{}
	for i := 0; i < len(num); i++ {
		c := num[i]
		for k > 0 && len(st) > 0 && st[len(st)-1] > c {
			st = st[:len(st)-1]
			k--
		}
		st = append(st, c)
	}
	for k > 0 && len(st) > 0 {
		st = st[:len(st)-1]
		k--
	}
	i := 0
	for i < len(st) && st[i] == '0' {
		i++
	}
	if i == len(st) {
		return "0"
	}
	return string(st[i:])
}
```

## 为什么这样写

十进制数的高位影响最大。如果前面有一个更大的数字，而后面来了更小数字，删除前面大的数字会让整体变小。删除额度有限，所以每次只在确定能变优时删除。

扫描结束后如果还有删除额度，说明剩余数字单调递增，只能从末尾删。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(n)$。

## 易错点

- 剩余 k 没用完。
- 前导零没有去掉。
- 删除后空串没有返回 `"0"`。
- 把 #402 和 #316 混淆：#316 还要保证每个字符出现一次。
