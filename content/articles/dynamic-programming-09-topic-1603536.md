# 最长上升子序列贪心优化：动态规划训练题解

LIS 先要会 $O(n^2)$ DP，再理解贪心 + 二分优化。优化不是换题，而是把“长度为 k 的递增子序列结尾越小越好”这个性质用数组维护起来。

一句话记法：**同样长度，结尾越小，未来越有机会变长。**

## O(n²) 状态定义

```text
dp[i] = 以 nums[i] 结尾的最长递增子序列长度
```

转移：

```text
dp[i] = max(dp[j] + 1), j < i 且 nums[j] < nums[i]
```

答案是 `max(dp)`，不是 `dp[n-1]`。

## Go 参考实现：贪心 + 二分

```go
func lengthOfLIS(nums []int) int {
	tails := []int{}
	for _, x := range nums {
		i := sort.Search(len(tails), func(i int) bool {
			return tails[i] >= x
		})
		if i == len(tails) {
			tails = append(tails, x)
		} else {
			tails[i] = x
		}
	}
	return len(tails)
}
```

## 为什么这样写

`tails[i]` 表示长度为 `i+1` 的递增子序列中，最小可能结尾值。它不一定对应某一条真实完整答案路径，但长度是真实有效的。

遇到 `x` 时：

- 如果 `x` 比所有结尾都大，可以接在最长序列后面，长度加一。
- 否则找到第一个 `>= x` 的结尾并替换，让这个长度的结尾更小。

严格递增用 `>=` 找替换位置；如果是不下降子序列，要改成 `>`。

## 复杂度

- DP 版本时间 $O(n^2)$、空间 $O(n)$。
- 贪心 + 二分时间 $O(n \log n)$、空间 $O(n)$。

## 易错点

- `dp[i]` 定义成前 i 个元素的 LIS，却又按“以 i 结尾”转移。
- 贪心数组 `tails` 误以为就是最终序列。
- 严格递增和非递减的二分条件混淆。
- 俄罗斯套娃信封没有先排序和处理同宽降序。
