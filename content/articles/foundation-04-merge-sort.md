# 归并排序：先让两半有序，再线性合并

归并排序把“排序整个区间”拆成三个明确动作：排序左半、排序右半、合并两个有序区间。递归函数的契约是：调用结束时，区间 `a[l..r)` 已有序。

## 合并为什么是线性的

左右两段都已有序时，只需比较两个段首：较小者一定是全局剩余元素中的最小值，把它放入临时数组并移动对应指针。每个元素只进入临时数组一次，因此合并成本为 $O(r-l)$。

```text
left  = [2, 7]
right = [3, 5]
依次取 2、3、5，最后接上 7
```

## 递归框架

```go
func sort(a, tmp []int, l, r int) {
    if r-l <= 1 {
        return
    }
    mid := l + (r-l)/2
    sort(a, tmp, l, mid)
    sort(a, tmp, mid, r)
    merge(a, tmp, l, mid, r)
}
```

统一使用左闭右开区间后，长度是 `r-l`，左右区间 `[l,mid)` 与 `[mid,r)` 无重叠也无遗漏。

## 复杂度推导

每层递归的所有区间总共处理 $n$ 个元素，递归树高度为 $\log n$，所以时间是 $O(n\log n)$。数组版本通常需要 $O(n)$ 辅助空间，递归栈另占 $O(\log n)$。

## 稳定性

当左右段首相等时先取左边元素，可以保持相等元素的原始相对顺序，因此归并排序可以稳定。若先取右边，相等元素会跨区间换序。

## 自顶向下与自底向上

自顶向下用递归拆分，代码直观；自底向上从长度 1 的段开始，两两合并成长度 2、4、8 的段，可以避免递归栈。链表归并还可以通过改指针完成合并，不必像数组一样复制元素。

## 易错点

- 递归区间没有缩小，导致无限调用。
- 临时数组写完后忘记复制回原区间。
- 一边耗尽后忘记接上另一边的剩余元素。
- 每层递归都新建大数组，造成不必要的分配；应复用一份辅助空间。
