# 前后缀乘积拆贡献：数组与字符串训练题解

前后缀拆贡献适合这样的题：每个位置的答案都依赖“左边的信息”和“右边的信息”。暴力做法会对每个位置重新扫左右两边；前后缀做法把这些重复扫描提前缓存或滚动维护。

典型例子是 #238 除自身以外数组的乘积：`ans[i] = 左侧所有数乘积 * 右侧所有数乘积`。不需要除法，也不用特殊讨论 0 的个数。

## 适用场景

当题目出现“除自身以外”“左边最大/右边最大”“左右两边和相等”“每个位置的贡献”时，优先考虑前后缀。

- 前缀和：快速知道某位置左侧总和。
- 后缀和：快速知道某位置右侧总和。
- 前缀积、后缀积：处理除自身以外乘积。
- 左侧最大值、右侧最大值：接雨水、糖果分发等题常用。

如果每个位置只依赖一边，单趟扫描就够；同时依赖两边时，通常需要两趟或两个辅助数组。

## 图解思路

```mermaid
flowchart LR
  A[从左到右] --> B[记录每个位置左侧贡献]
  B --> C[从右到左]
  C --> D[合并右侧贡献]
  D --> E[得到当前位置答案]
```

最稳定的写法是两趟：

- 第一趟：`ans[i]` 先放左侧贡献。
- 第二趟：用一个滚动变量 `suffix` 表示右侧贡献，并乘回 `ans[i]`。
- 更新顺序：先合并当前位置，再把当前元素纳入后缀。

这个顺序能自然表达“除自身以外”：当前位置的元素不能参与自己的左侧或右侧贡献。

## 手写步骤

1. 明确当前位置答案由哪两部分组成。
2. 从左到右预处理左侧贡献，注意不包含当前元素。
3. 从右到左预处理或滚动维护右侧贡献，注意不包含当前元素。
4. 在当前位置合并左右贡献。
5. 检查乘积、和或最大值的初始值是否符合空前缀/空后缀含义。

## Go 参考骨架

```go
func productExceptSelf(nums []int) []int {
	ans := make([]int, len(nums))
	prefix := 1
	for i, x := range nums {
		ans[i] = prefix
		prefix *= x
	}
	suffix := 1
	for i := len(nums) - 1; i >= 0; i-- {
		ans[i] *= suffix
		suffix *= nums[i]
	}
	return ans
}
```

## Rust 参考骨架

```rust
pub fn pivot_index(nums: Vec<i32>) -> i32 {
    let total: i32 = nums.iter().sum();
    let mut left_sum = 0;
    for (i, &x) in nums.iter().enumerate() {
        if left_sum == total - left_sum - x {
            return i as i32;
        }
        left_sum += x;
    }
    -1
}
```

## 为什么这样写

以 #238 为例，不能简单用总乘积除以 `nums[i]`，因为数组可能有 0，题目也常限制不能用除法。前后缀积刚好避开这个问题。

第一趟结束后，`ans[i]` 是 `nums[0..i)` 的乘积；第二趟从右往左时，`suffix` 是 `nums(i..n)` 的乘积。两者相乘，就是除自身以外的全部元素乘积。

以 #724 寻找数组中心下标为例，它也是相同思想，只是右侧贡献可以由 `total - left_sum - nums[i]` 直接算出来，不一定要单独建后缀数组。

## 复杂度

- 时间复杂度：$O(n)$，通常两趟扫描。
- 空间复杂度：如果答案数组不计入额外空间，滚动后缀变量可以做到 $O(1)$；显式前后缀数组是 $O(n)$。

## 易错点

- 前缀或后缀先乘当前元素，导致把自己也算进答案。
- 乘积初始值写成 `0`，空乘积应该是 `1`。
- 中心下标题忘记排除当前元素，右侧和应为 `total - left_sum - nums[i]`。
- 接雨水类题中，当前位置水量由左右最大值的较小者决定，不是较大者。

## 练习顺序

建议按这个顺序刷：#724, #238, #42, #135, #1991, #152。

先用 #724 和 #238 练基础前后缀，再做 #42/#135 体会“左右约束取较小或较大”的贡献合并，最后用 #152 感受乘积里负数带来的状态变化。
