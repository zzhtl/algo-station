# 连续相同段压缩统计：数组与字符串精讲

## 一句话总结

段的左右边界一旦确定，段内每个位置的字符相同，答案只依赖段长和段外连接关系。

## 识别信号

遇到连续相同字符、压缩字符串、删除重复段、统计同字符子串。

对「连续相同段压缩统计」来说，识别信号不是关键词本身，而是它们都在要求你在一段连续结构里保留足够信息，让窗口、指针或扫描线每推进一步都能解释答案为什么没有被漏掉。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 指针与区间锚点

- 本篇围绕「连续相同段压缩统计」训练：段的左右边界一旦确定，段内每个位置的字符相同，答案只依赖段长和段外连接关系。
- 数组题的核心不是多写循环，而是让每个下标只承担一种角色：左边界、右边界、候选值或答案落点。
- 识别信号里的“遇到连续相同字符、压缩字符串、删除重复段、统计同字符子串”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #443、#38、#1759、#1957 时，复盘「边界移动、候选淘汰、答案写入」是否还能用同一套解释串起来。

## 推导步骤

1. 固定左端点，右端点扩到第一个不同字符。
2. 用 `[l,r)` 表示一段。
3. 按段长计算贡献或写回结果。
4. 处理最后一段。

## 动手前的边界清单

- 空数组、单元素、全相等、严格递增或递减都要先手算一遍。
- 对照「连续相同段压缩统计」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #443、#38、#1759、#1957 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 忘记处理最后一段。
- 写回压缩结果时读写指针互相覆盖。
- 把字符段和数字段边界混在一起。
