# 换根 DP 拆上下贡献：动态规划精讲

## 一句话总结

节点答案由子树内贡献和父侧贡献合并而成，换根时只需局部调整。

## 识别信号

树中距离之和、最小高度树、统计可能根数。

对「换根 DP 拆上下贡献」来说，识别信号不是关键词本身，而是它们都在要求你把重复子问题压成状态，并保证每一次转移只从已经定义清楚的旧状态来。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 状态转移锚点

- 本篇围绕「换根 DP 拆上下贡献」训练：节点答案由子树内贡献和父侧贡献合并而成，换根时只需局部调整。
- 状态含义要先于代码确定；如果一句话说不清 dp[i] 表示什么，转移大概率会写散。
- 识别信号里的“树中距离之和、最小高度树、统计可能根数”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #834、#310、#2581 时，复盘「状态定义、转移来源、遍历顺序」是否还能用同一套解释串起来。

## 推导步骤

1. 第一次 DFS 算 size 和子树答案。
2. 第二次从父到子换根。
3. 用公式移除子贡献并加入父侧贡献。

## 写转移前的边界清单

- 初始状态、不可能状态、容量或下标为 0、答案取 max 还是求和要分清。
- 对照「换根 DP 拆上下贡献」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #834、#310、#2581 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 换根公式符号错。
- 无向树没有 parent 防回走。
- 把边贡献和点贡献混淆。
