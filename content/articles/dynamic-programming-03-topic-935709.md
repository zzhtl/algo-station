# 背包容量倒序更新：动态规划训练题解

0/1 背包的核心限制是：每件物品最多选一次。一维滚动时，容量倒序更新就是为了守住这个限制。

一句话记法：**0/1 背包倒序容量，确保读到的是上一轮状态。**

## 适用场景

- 分割等和子集。
- 最后一块石头重量 II。
- 目标和转换成选正负号。
- 每个元素最多使用一次的可行性或最值问题。

只要元素不能重复用，就先想到 0/1 背包。

## 状态定义

可行性版本：

```text
dp[j] = 是否能用已处理物品凑出容量 j
```

最值版本：

```text
dp[j] = 容量不超过 j 时能得到的最大价值
```

## Go 参考实现：分割等和子集

```go
func canPartition(nums []int) bool {
	sum := 0
	for _, x := range nums {
		sum += x
	}
	if sum%2 == 1 {
		return false
	}
	target := sum / 2
	dp := make([]bool, target+1)
	dp[0] = true
	for _, x := range nums {
		for j := target; j >= x; j-- {
			dp[j] = dp[j] || dp[j-x]
		}
	}
	return dp[target]
}
```

## 为什么倒序

假设正序更新：

```text
for j := x; j <= target; j++
```

那么 `dp[j-x]` 可能已经在本轮因为当前物品 `x` 被更新过。再用它更新 `dp[j]`，就等于当前物品被用了两次。

倒序时，`dp[j-x]` 还保持上一轮结果，所以每个物品最多贡献一次。

## 复杂度

- 时间复杂度：$O(n \cdot target)$。
- 空间复杂度：$O(target)$。

## 易错点

- 容量正序更新，变成完全背包。
- `dp[0]` 没初始化为 true。
- 总和为奇数还继续做 DP。
- 目标和 #494 没先做代数转换，状态设计复杂化。
