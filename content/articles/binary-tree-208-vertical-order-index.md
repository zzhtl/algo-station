# 树节点坐标化遍历：二叉树精讲

## 一句话总结

树结构可以映射到二维坐标：左子列减一，右子列加一，深度加一。

## 识别信号

垂序遍历、二叉树宽度、右视图、列聚合。

对「树节点坐标化遍历」来说，识别信号不是关键词本身，而是它们都在要求你把整棵树的问题拆成当前节点能向父节点交付什么，以及当前节点如何合并左右子树的信息。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 递归返回值锚点

- 本篇围绕「树节点坐标化遍历」训练：树结构可以映射到二维坐标：左子列减一，右子列加一，深度加一。
- 递归函数的返回值必须只表达对子树已经完成的承诺，不能夹带依赖父节点才知道的答案。
- 识别信号里的“垂序遍历、二叉树宽度、右视图、列聚合”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #987、#314、#662、#199 时，复盘「返回值含义、全局答案更新点、空节点处理」是否还能用同一套解释串起来。

## 推导步骤

1. BFS/DFS 携带 row/col。
2. 按 col 聚合。
3. 需要稳定顺序时排序 row 和值。
4. 宽度用完全二叉树编号。

## 写递归前的边界清单

- 空树、单节点、只有左子树、只有右子树、重复值都要单独过。
- 对照「树节点坐标化遍历」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #987、#314、#662、#199 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 编号溢出不归一。
- 同列同层排序规则漏掉。
- DFS 顺序不能天然代表层序。
