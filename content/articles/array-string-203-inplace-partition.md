# 原地分区保持不变量：数组与字符串精讲

## 一句话总结

任何时刻 `[0,slow)` 都是已经放好的元素，`fast` 只负责扫描未知区域。

## 识别信号

移除元素、颜色分类、奇偶分离、移动零、按条件稳定写入。

对「原地分区保持不变量」来说，识别信号不是关键词本身，而是它们都在要求你在一段连续结构里保留足够信息，让窗口、指针或扫描线每推进一步都能解释答案为什么没有被漏掉。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 指针与区间锚点

- 本篇围绕「原地分区保持不变量」训练：任何时刻 `[0,slow)` 都是已经放好的元素，`fast` 只负责扫描未知区域。
- 数组题的核心不是多写循环，而是让每个下标只承担一种角色：左边界、右边界、候选值或答案落点。
- 识别信号里的“移除元素、颜色分类、奇偶分离、移动零、按条件稳定写入”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #27、#75、#905、#922、#283 时，复盘「边界移动、候选淘汰、答案写入」是否还能用同一套解释串起来。

## 推导步骤

1. 定义已处理区间。
2. 扫描每个元素。
3. 满足条件就写到 slow 并推进。
4. 不满足条件只推进 fast。
5. 最后截断或填充。

## 动手前的边界清单

- 空数组、单元素、全相等、严格递增或递减都要先手算一遍。
- 对照「原地分区保持不变量」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #27、#75、#905、#922、#283 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 需要稳定顺序时误用左右交换。
- slow/fast 初始区间含义不清。
- 写入后忘记推进 slow。
