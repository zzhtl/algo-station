# 括号匹配的类型栈：栈与队列训练题解

括号有效性不是只看数量是否相等，还要看嵌套顺序和类型。最近打开的括号必须最先闭合，这正是栈的后进先出。

一句话记法：**左括号入栈，右括号必须匹配栈顶。**

## 适用场景

- 判断括号字符串是否合法。
- 多种括号类型。
- 最近打开的结构要最先关闭。
- HTML/XML 标签匹配也有类似思想。

## 不变量

- 栈中保存尚未匹配的左括号。
- 栈顶是当前最内层括号。
- 遇到右括号时，栈不能为空且类型必须匹配。
- 扫描结束后，栈必须为空。

## Go 参考实现

```go
func isValid(s string) bool {
	pairs := map[byte]byte{
		')': '(',
		']': '[',
		'}': '{',
	}
	st := []byte{}
	for i := 0; i < len(s); i++ {
		ch := s[i]
		if ch == '(' || ch == '[' || ch == '{' {
			st = append(st, ch)
			continue
		}
		if len(st) == 0 || st[len(st)-1] != pairs[ch] {
			return false
		}
		st = st[:len(st)-1]
	}
	return len(st) == 0
}
```

## 为什么这样写

字符串 `"([)]"` 中括号数量是对的，但顺序不对。扫描到 `)` 时，最近未闭合的是 `[`，所以应该失败。栈顶正好表达“最近未闭合”的结构。

最后栈不为空，说明还有左括号没被关闭，也必须失败。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(n)$。

## 易错点

- 只统计三种括号数量，不检查嵌套顺序。
- 遇到右括号时没判断栈空。
- 扫描结束忘记检查栈是否为空。
- 匹配表方向写反。
