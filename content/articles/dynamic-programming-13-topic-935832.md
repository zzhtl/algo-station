# 买卖股票持仓状态：动态规划训练题解

股票题不要按“哪天买哪天卖”枚举。更稳定的建模是看每天结束时的状态：手里有没有股票。

一句话记法：**`hold` 表示持有，`cash` 表示不持有。**

## 基础无限交易

```go
func maxProfit(prices []int) int {
	hold := -prices[0]
	cash := 0
	for i := 1; i < len(prices); i++ {
		price := prices[i]
		newCash := max(cash, hold+price)
		newHold := max(hold, cash-price)
		cash, hold = newCash, newHold
	}
	return cash
}
```

## 各变体怎么加约束

| 题型 | 变化 |
| --- | --- |
| 只能交易一次 | 买入只能来自初始现金 0 |
| 无限次交易 | `cash` 和 `hold` 互相转移 |
| 含手续费 | 买入或卖出时扣 fee |
| 含冷冻期 | 买入要来自前两天的 `cash` |
| 最多 k 次 | 状态加交易次数维度 |

## 为什么这样写

每天结束后不持有股票，可能是昨天也不持有，也可能是今天卖了。每天结束后持有股票，可能是昨天也持有，也可能是今天买了。

这比枚举买卖日更接近题目过程，也容易扩展到手续费和冷冻期。

## 复杂度

- 基础版本时间 $O(n)$、空间 $O(1)$。
- 最多 k 次交易通常是 $O(nk)$。

## 易错点

- 最后返回 `hold`，但持有股票收益还没兑现。
- 手续费买入和卖出都扣。
- 冷冻期买入时仍然用昨天的 `cash`。
- 交易次数到底在买入时加还是卖出时加没有统一。
