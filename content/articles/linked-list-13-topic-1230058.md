# 旋转链表成环再断开：链表训练题解

旋转链表的本质是把尾部 `k` 个节点搬到前面。与其逐个移动，不如先把链表首尾连成环，再在正确位置断开。

一句话记法：**先成环，再走到新尾，断开就是答案。**

## 适用场景

- 右旋链表。
- 需要把链表尾部一段接到头部。
- `k` 可能远大于链表长度。

如果链表为空、只有一个节点，或 `k % n == 0`，可以直接返回原头。

## 图解思路

```text
1 -> 2 -> 3 -> 4 -> 5, k = 2

成环: 1 -> 2 -> 3 -> 4 -> 5
      ^                   |
      |___________________|

新尾是第 n-k 个节点: 3
断开后: 4 -> 5 -> 1 -> 2 -> 3
```

新头是新尾的下一个节点。

## 不变量

- `n` 是链表长度。
- 右旋 `k` 等价于右旋 `k % n`。
- 新尾是原链第 `n - k % n` 个节点。
- 成环后必须断开新尾的 `Next`。

## 手写步骤

1. 处理空链和单节点。
2. 遍历求长度，同时找到尾节点。
3. `k %= n`，若为 0 直接返回。
4. 尾节点接回 head 成环。
5. 从 head 走 `n-k-1` 步到新尾。
6. 记录新头，断开新尾。

## Go 参考实现

```go
func rotateRight(head *ListNode, k int) *ListNode {
	if head == nil || head.Next == nil || k == 0 {
		return head
	}

	n := 1
	tail := head
	for tail.Next != nil {
		tail = tail.Next
		n++
	}

	k %= n
	if k == 0 {
		return head
	}

	tail.Next = head
	steps := n - k - 1
	newTail := head
	for i := 0; i < steps; i++ {
		newTail = newTail.Next
	}
	newHead := newTail.Next
	newTail.Next = nil
	return newHead
}
```

## 为什么这样写

右旋 `k` 次后，原链表倒数第 `k` 个节点会变成新头。等价地，新尾是它前面的节点，也就是第 `n-k` 个节点。由于 `k` 可能超过长度，先取模可以去掉整圈旋转。

成环让“尾接头”变成一步操作，然后只需要在新尾处断开。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(1)$。

## 易错点

- 忘记 `k %= n`，多走很多无意义步。
- 成环后忘记断开，返回有环链表。
- 新尾步数写成 `n-k`，导致落到新头。
- 空链表时访问 `head.Next`。
