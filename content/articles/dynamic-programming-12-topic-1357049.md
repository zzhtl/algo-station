# 数字解码按前缀转移：动态规划训练题解

数字解码的状态适合定义在前缀上：`dp[i]` 表示前 `i` 个字符的解码方法数。转移只看最后一位和最后两位。

一句话记法：**单字符合法加 `dp[i-1]`，双字符合法加 `dp[i-2]`。**

## 状态定义

```text
dp[i] = s[0:i] 的解码方法数
```

边界：

```text
dp[0] = 1
```

空前缀算一种“什么都不选”的方式，方便转移。

## Go 参考实现

```go
func numDecodings(s string) int {
	n := len(s)
	dp := make([]int, n+1)
	dp[0] = 1
	for i := 1; i <= n; i++ {
		if s[i-1] != '0' {
			dp[i] += dp[i-1]
		}
		if i >= 2 {
			x := int(s[i-2]-'0')*10 + int(s[i-1]-'0')
			if x >= 10 && x <= 26 {
				dp[i] += dp[i-2]
			}
		}
	}
	return dp[n]
}
```

## 为什么这样写

最后一步只有两种可能：

- 最后一位单独解码，比如 `...3`。
- 最后两位一起解码，比如 `...12`。

`0` 不能单独解码，所以 `s[i-1] == '0'` 时不能加 `dp[i-1]`。但 `10` 和 `20` 是合法的双字符。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(n)$，可压缩到两个变量。

## 易错点

- 把 `0` 当成单独字符解码。
- `06` 被误判成合法双字符。
- `dp[0]` 没初始化为 1。
- 下标 `i` 和字符串位置 `i-1` 混淆。
