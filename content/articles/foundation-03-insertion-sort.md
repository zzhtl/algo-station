# 插入排序：维护不断扩大的有序区

插入排序模拟整理手牌：左边始终有序，每次从右边取出一个 `key`，在有序区中为它找到位置。核心不变量是：处理下标 `i` 前，`a[0..i)` 已经有序。

## 为什么先移动再插入

假设数组为 `[2, 5, 4, 6]`，当前 `key=4`。先保存 4，把比它大的 5 向右移动，空位落在 2 和 5 之间，再把 4 写入空位。

```go
for i := 1; i < len(a); i++ {
    key := a[i]
    j := i - 1
    for j >= 0 && a[j] > key {
        a[j+1] = a[j]
        j--
    }
    a[j+1] = key
}
```

如果每比较一次就交换，也能得到正确结果，但写入次数更多。“保存 key → 整体右移 → 一次写回”更接近算法本质。

## 正确性

- 初始时只有 `a[0]`，天然有序。
- 一轮中把所有大于 `key` 的元素右移，再把 `key` 放进唯一空位，有序区扩展一个元素。
- 当 `i == n`，整个数组就是有序区。

## 稳定性

比较条件必须是 `a[j] > key`，而不是 `>=`。遇到相等元素时停止移动，原来靠前的元素仍靠前，所以插入排序是稳定排序。稳定性在“先按次关键字排序、再按主关键字稳定排序”等场景很重要。

## 复杂度

- 最坏情况：逆序数组，每轮移动整个有序区，$O(n^2)$。
- 最好情况：数组已有序，每轮只比较一次，$O(n)$。
- 额外空间：$O(1)$。

因此插入排序适合数据量小或近乎有序的局部数组。工程排序常在分区变小后切换到插入排序，减少递归和常数开销。

## 易错点

- 没先保存 `key`，右移时覆盖了它。
- `j` 退出后把 `key` 写到 `j`，正确位置应为 `j+1`。
- 使用 `>=`，无意中破坏稳定性。
- 把“有序区”误写成“已经处在最终位置”；插入排序中有序区元素后续仍可能右移。
