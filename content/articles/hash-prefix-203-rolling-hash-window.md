# 滚动哈希窗口匹配：哈希与前缀精讲

## 一句话总结

窗口右移时删除左端贡献、加入右端字符，哈希值代表当前窗口。

## 识别信号

重复 DNA、最长重复子串、字符串匹配、不同回文子序列计数。

对「滚动哈希窗口匹配」来说，识别信号不是关键词本身，而是它们都在要求你把重复查询改成一次记录，让“以前见过什么”和“当前还缺什么”可以 O(1) 对上。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

- 本篇围绕「滚动哈希窗口匹配」训练：窗口右移时删除左端贡献、加入右端字符，哈希值代表当前窗口。
- 哈希表里保存的必须是可复用的证据，而不是碰巧能过样例的中间值。

1. 预处理幂。
2. 维护窗口哈希。
3. 同哈希时必要时回查原串。
4. 用集合记录出现状态。

## 建表前的边界清单

- 重复值、负数、前缀为 0、同一个键多次出现要单独验。
- 对照「滚动哈希窗口匹配」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #28、#187、#1044、#1316 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 模数或 base 太小导致冲突。
- 窗口长度为 0/超过字符串。
- 字符映射不统一。
