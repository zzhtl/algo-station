# 入度拓扑排序：搜索与图论精讲

## 一句话总结

入度为 0 表示当前没有未满足前置条件，可以先处理。

## 识别信号

课程表、课程表 II、最小高度树、项目分组排序。

对「入度拓扑排序」来说，识别信号不是关键词本身，而是它们都在要求你先把题目里的对象建成状态和边，再决定用 BFS、拓扑、最短路还是连通性工具推进。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 状态建图锚点

- 本篇围绕「入度拓扑排序」训练：入度为 0 表示当前没有未满足前置条件，可以先处理。
- visited、距离或并查集保存的是图上的事实，不能只按原始节点编号粗略标记。
- 识别信号里的“课程表、课程表 II、最小高度树、项目分组排序”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #207、#210、#310、#1203 时，复盘「状态维度、边的语义、访问标记时机」是否还能用同一套解释串起来。

## 推导步骤

1. 建图和入度。
2. 所有入度 0 入队。
3. 弹出节点并降低邻居入度。
4. 统计处理数量。

## 入队松弛前的边界清单

- 孤立点、自环、重边、不可达、起终点相同都要考虑。
- 对照「入度拓扑排序」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #207、#210、#310、#1203 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 边方向建反。
- 重复边导致入度重复。
- 有环时没有检查处理数量。
