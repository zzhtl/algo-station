# 状态压缩 DP 枚举集合：动态规划精讲

## 一句话总结

一个整数的每一位表示一个元素是否已被选择，集合转移就是置位或清位。

## 识别信号

划分 K 子集、访问所有节点最短路、最小团队、帽子分配。

对「状态压缩 DP 枚举集合」来说，识别信号不是关键词本身，而是它们都在要求你把重复子问题压成状态，并保证每一次转移只从已经定义清楚的旧状态来。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 状态转移锚点

- 本篇围绕「状态压缩 DP 枚举集合」训练：一个整数的每一位表示一个元素是否已被选择，集合转移就是置位或清位。
- 状态含义要先于代码确定；如果一句话说不清 dp[i] 表示什么，转移大概率会写散。
- 识别信号里的“划分 K 子集、访问所有节点最短路、最小团队、帽子分配”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #698、#847、#1125、#1434 时，复盘「状态定义、转移来源、遍历顺序」是否还能用同一套解释串起来。

## 推导步骤

1. 定义 mask。
2. 枚举下一个未选元素。
3. 更新新 mask。
4. 必要时带当前位置或当前桶和。

## 写转移前的边界清单

- 初始状态、不可能状态、容量或下标为 0、答案取 max 还是求和要分清。
- 对照「状态压缩 DP 枚举集合」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #698、#847、#1125、#1434 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- n 太大仍用 bitmask。
- 重复状态没有记忆化。
- 子集枚举漏掉空集或全集。
