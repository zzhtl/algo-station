# 树形 DP 的选与不选：二叉树精讲

## 一句话总结

父子关系上的约束要通过状态拆开，避免父节点决策污染子树。

## 识别信号

打家劫舍 III、监控二叉树、最长交错路径。

对「树形 DP 的选与不选」来说，识别信号不是关键词本身，而是它们都在要求你把整棵树的问题拆成当前节点能向父节点交付什么，以及当前节点如何合并左右子树的信息。读题时先把样例拆成“已经确认”“仍在候选”“答案落点”三块，再决定用哪种结构推进。

## 递归返回值锚点

- 本篇围绕「树形 DP 的选与不选」训练：父子关系上的约束要通过状态拆开，避免父节点决策污染子树。
- 递归函数的返回值必须只表达对子树已经完成的承诺，不能夹带依赖父节点才知道的答案。
- 识别信号里的“打家劫舍 III、监控二叉树、最长交错路径”要能和代码里的每一次推进对应起来，否则说明你只记住了模板，没有抓住约束。
- 做完 #337、#968、#1372 时，复盘「返回值含义、全局答案更新点、空节点处理」是否还能用同一套解释串起来。

## 推导步骤

1. 后序递归。
2. 对子节点取状态。
3. 当前选时子节点必须不选。
4. 当前不选时子节点可选可不选。

## 写递归前的边界清单

- 空树、单节点、只有左子树、只有右子树、重复值都要单独过。
- 对照「树形 DP 的选与不选」标明答案在遍历中何时被写入，避免把中间状态误当最终结论。
- 相关题 #337、#968、#1372 只保留一个变化点：输入形态、约束或返回值发生了什么改变？
- 若代码需要第二遍、回溯或压缩状态，说明第一遍保留下来的信息刚好够用。

## 易错点

- 贪心选大节点导致局部最优。
- 状态含义不互斥。
- 监控题状态不完整。
