# 路径简化丢弃空段：栈与队列训练题解

Unix 路径简化可以看成处理目录栈。每进入一个普通目录就入栈，遇到 `..` 就回到上一层。

一句话记法：**普通目录入栈，`..` 弹栈，`.` 和空字符串跳过。**

## 适用场景

- 简化绝对路径。
- 处理连续斜杠。
- 处理当前目录和上级目录。

题目是绝对路径，所以栈弹空后再遇到 `..` 仍然停在根目录。

## 不变量

- 栈中保存从根到当前目录的有效路径段。
- 空段来自连续 `/`，忽略。
- `.` 表示当前目录，忽略。
- `..` 表示上一级，栈非空时弹出。

## Go 参考实现

```go
func simplifyPath(path string) string {
	parts := strings.Split(path, "/")
	st := []string{}
	for _, part := range parts {
		if part == "" || part == "." {
			continue
		}
		if part == ".." {
			if len(st) > 0 {
				st = st[:len(st)-1]
			}
		} else {
			st = append(st, part)
		}
	}
	return "/" + strings.Join(st, "/")
}
```

## 为什么这样写

路径 `/a//b/./../c/` 切分后会出现空段和特殊段。真正保留的目录变化是：

```text
a 入栈
b 入栈
. 忽略
.. 弹出 b
c 入栈
结果 /a/c
```

栈正好表达“当前路径从根到当前位置的目录序列”。

## 复杂度

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(n)$。

## 易错点

- 连续斜杠产生的空段没有忽略。
- 栈空时遇到 `..` 仍然弹栈。
- 返回结果末尾多一个 `/`。
- Go 代码需要引入 `strings`。
